## 为什么要有 v3

2026-05-15 到 2026-05-18 之间出了三轮复盘，它们把「一个小团队还能守得住的动作」重新定义了一遍：

- **v2 的智能体闭环结论**点了四个早已发表的闭环 VC FM 系统的名（「Closing the loop」、VCHarness、CellForge、BioLab），结论是：闭环这套架构已经谈不上新了——剩下能拼的，是*应用 + 纪律*。
- **101 巡礼**把领域层面的心智模型挑明了：底料（第一阶图谱、第二阶扰动数据、纵向病人面板）才是如今每一个有意思的问题真正绕不开的地方。
- **邻近形态 + n=1 那篇**把同一套复盘搬到三种后续项目形态上（多细胞性 FM、跨癌种 anti-PD1、n=1 个性化），结果同一个套路又出现了三遍：闭环也好、模型也好，要么已发表、要么已商品化，唯一守得住的新意都活在*上游*的底料决策里——湿实验图谱的产出、评测纪律、纵向采样设计。

v2 那篇把范围收到三条（架构、适配器、互操作）就其本身而言依然没错——但它没说每一条到底在守哪块底料，而且它把 v1 里那条智能体包装继续摆着，可闭环那批文献到现在早就把它做烂了。v3 把这两点都补上。

## 重排的内核——底料才是护城河

到了 2026 年：闭环是大路货，骨干是大路货，架构基本是大路货，连包装也越来越是大路货。大 FM 实验室和那些钱多的智能体公司，这些东西它们要么追平要么超过。它们*追不上*的，是底料——一个特定的图谱、一支有特权的队列、一块临床参考面板、一种社区共治的格式、一份别人压根没想到要去编码的生物学直觉。下面每一条诚实的小实验室切入点，都是按「它在守哪块底料」来定义的，而不是按它用了什么技术。一个 LoRA 适配器算不上护城河；*一个训在你比谁都更懂的下游任务上的 LoRA 适配器*才算。

这套重排，正是 v2 那篇最后落到「拼的是应用 + 纪律，不是方法」的原因，也是邻近形态复盘得出**每一种邻近形态里,护城河都在模型上游**的原因。这篇剩下的内容，就是把这条结论摊成六个切入点。

## 各切入点的资源门槛

下面六条在资源量级上根本不对等。两条是货真价实的「一个工程师 + 一块卡」；两条是披着小实验室外衣的机构级动作；还有一条更需要的是临床合作者而不是 GPU。下面这张表先把每条的资源门槛点出来，*放在*各条详述里的底料 / 技能细节之前——先读表，再拿它去对照后面的小节。

| 切入点 | 算力门槛 | 数据 | 其他资源门槛 | 资源裁定 |
|---|---|---|---|---|
| 1 — 架构归纳偏置 | ~$5–20K 云费 + 1 个 A100/A6000 节点 | CELLxGENE Census + sc-Arena / PerturBench 基准（公开） | 6–12 个月工程师时间 | **临界。** 预印本够得着；SOTA 旗号够不着——真要收敛到一个非 BERT 架构，迭代轮次实际需要的算力大约是头条数字的 10 倍。 |
| 2 — 冻结骨干 + 适配器 | $0–2K，本地硬件上一个 GPU-周 | Replogle、Tahoe-100M、GDSC、PRISM、CCLE——全公开 | 一个工程师 × 6 个月 | **最贴合。** 唯一一条头条数字（「1–10 GPU-天」）货真价实的切入点。 |
| 3 — 以病人为锚的投影 | ~1 GPU-天，微不足道 | Sade-Feldman / Jerby-Arnon / Bassez / Bi 图谱公开；病人内验证样本不公开 | **IRB + 临床合作者 + 活检 + DUA，往往还得有医院身份** | 算力很小；**临床合作者这个名额才是整道护城河**。没有它，这条切入点就关着。 |
| 4 — 特权语料持续预训练 | $20–100K（Geneformer-CLcancer 量级） | **按定义就是独占** | 整理基础设施 + 机构身份 | **机构级，非小实验室。** 「特权语料」不过是「机构准入」的另一种说法。 |
| 5 — 宽松许可重实现 | $5–20K（xVERSE 级）到 $50–200K（Hibou / TranscriptFormer / COMPASS 级） | 公开，但协调统一很费钱 | 发布工程 + 持续维护 | **部分贴合。** 只在小模型那一端可行——大多数被锁起来的目标，正是因为它复现起来很贵才被锁。 |
| 6 — 互操作基础设施 | ~$0，一台笔记本就能跑 | 常常什么都不需要 | 多年持续的工程师时间 + 社区 / 治理工作 | **和 W2 并列最贴合。** 瓶颈是年头，不是钱。 |

对着这张表读：在资源量级上无可争议地适合小实验室的，只有**第 2 条和第 6 条**。**第 1 条**有一个 GPU 节点加一笔不大的云预算就能过线。**第 3 条**更需要临床合作者而非算力——v3 把它当小实验室切入点来讲，只在那个名额被填上的前提下才算诚实。**第 4、5 条**是披着小实验室外衣的机构级动作，尽管下面各条的正文把它们写得跟第 1、2、6 条好像可以平起平坐。「底料即护城河」这套重排，恰恰就是为了*点破*这一点：护城河都在模型上游，而上游那些资源（数据、临床准入、持续工程）大多不是一个小实验室拿一块 GPU 加一年时间就能买来的。

## 切入点 1 — 架构归纳偏置

**底料：你的生物学直觉。**

一个专为生物学量身定做的全新架构组件或完整骨干——而不是从 NLP 那边搬过来改改的 transformer。大 FM 实验室卡在 BERT 的路径依赖里出不来；你的防线，就是那一条你愿意在架构层面把它编码进去的生物学事实。

**代表作**（基本沿用 v2）：**xVERSE**（Jiang & Xie, bioRxiv 2026.04）——转录组学原生的非 LM 架构，相比 scGPT / Geneformer / scFoundation / UCE 提升 11–34%；**TxPert**（Wenkel et al., *Nat Biotech* 2026）——把多知识图谱先验注进扰动 transformer；**ESM-3 的 SE(3)-不变 token**；**AlphaGenome 的轨迹预测目标**；以及面向基因组底料的状态空间骨干（Hyena / Mamba / Evo2）。

**底料自测：**问一句，你的架构编码了哪种对称性、先验或生物学结构，是普通 transformer 没有的。如果你能一句话说清楚，这就是条真切入点。说不清楚，那你不过是又发了一个 BERT。

**技能：**PyTorch/JAX + 有主见的生物学。算力其实不大（个位数 GPU-周）。

## 切入点 2 — 冻结骨干 + 适配器

**底料：一个你比谁都更懂的下游任务。**

一个 LoRA / Houlsby 适配器 / 专用头（占骨干参数的 0.1–5%），专门去修预训练 FM 的某一种具体失效。2025 年那场线性基线清算，让这一招成了整个领域杠杆率最高的动作。

**代表作：sc-FM Perturbation Adapter**（ICLR 2026）——一个 <1% 的药物条件适配器，赢过线性加性基线；**PertAdapt**（bioRxiv 2025）——带基因级结构的条件敏感适配器；**scDCA**（ICLR 2026）——挂在冻结 sc-FM 上的药物条件适配器；细胞类型专用的分类头；sc-FM 与空间 FM 嵌入空间之间的跨模态适配器；以及病理 FM（CONCH、TITAN、CHIEF）上的注意力-MIL 聚合器。

**底料自测：**说出那个下游任务，再讲清楚*为什么你对任务的理解能压过隔壁团队的超参搜索*。要是答案是「我就是把头调得更好」，那是一篇论文，不是护城河。

**技能：**PyTorch + 对任务的深刻理解。算力是每次实验 1–10 GPU-天。

## 切入点 3 — 以病人为锚的投影（新增）

**底料：一个特定的临床参考图谱 + 纵向采样纪律。**

这条在 v1、v2 里都不存在。它是 [邻近形态那篇](adjacent-project-shapes-and-n-of-1.md) 里那次 n=1 微调复盘明确点出来的，而且它确实是个小团队动作：拿一份已发表的应答者图谱（Sade-Feldman 黑色素瘤、Jerby-Arnon、Bassez 乳腺、Bi 肾癌），用 **scArches**（[Lotfollahi et al., *Nat Biotech* 2021](https://www.nature.com/articles/s41587-021-01001-7)）把新病人数据投影进去，再接一个冻结 FM 的特征提取器 + 队列上训出来的 logistic / GBM 头，最后把成果做成一个 HuggingFace Space 或一件临床研究工具发出去。

它之所以成立：每一篇闭环 VC 论文都默认你能微调骨干，而每一句 n=1 个性化的主张都会一头撞上灾难性遗忘那堵墙。scArches 式的投影把这两道坎都绕开了——骨干冻结、FM 冻结、队列上训头、*逐病人推断*。「模型」在病人之间一动不动；变的是每个病人在图谱里走出的那条投影轨迹。这是 2026 年文献里唯一诚实的 n=1 动作，而几乎没人把它做成一件*产品化*的小团队工件。

**代表作：scArches** 本身及其衍生；**Tempus IPS** 和 **PurIST 胰腺**那套套路（队列上训好头，逐病人查询）；以及在留一癌种评测下，病人专属的 **Virchow2** + 注意力-MIL 聚合器跑 H&E 做 ICI 应答（这正是 [邻近形态那篇](adjacent-project-shapes-and-n-of-1.md) 点出的那个守得住的小生境）。

**底料自测：**在写代码*之前*就把参考图谱 + 留出评测队列说清楚。它们要是不存在、或者你拿不到，这条切入点就不向你敞开。

**技能：**对 scArches/scvi-tools 的熟练，外加一个能提供病人内纵向样本来做验证的临床合作者。算力是每次投影一个 GPU-天。

## 切入点 4 — 特权语料持续预训练

**底料：你能拿到、别人拿不到的数据。**

拿一个开放权重的 FM，在一份只有你能攒齐的、严格整理过的领域语料上接着预训练，再把新 checkpoint 发出去。这是 v1 里的第 3 条。

**代表作：Geneformer V2-104M_CLcancer**——通用版 Geneformer 在 1400 万个整理过的癌症细胞上持续预训练而成；尽管小了 3 倍，在大多数癌症下游任务上却追平甚至超过 316M 的通用模型。

**加硬版的底料自测：**你的语料凭什么*独占*？要是答案是「我们从 CELLxGENE Census 拉的」，这条切入点就不归你——大 FM 实验室早就在 Census 上预训练过了。只有当你手里有一份特权语料（某个机构的罕见病队列、某个组织专属的档案、一批别人碰不到的临床试验数据），这条路才向你敞开。

**技能：**数据工程（语料整理才是大头）+ 足够熟的 HuggingFace 功底，能干净利落地续训。算力随语料规模在 10–100 GPU-天之间。

## 切入点 5 — 宽松许可重实现

**底料：发布工程 + 许可清晰度，这是那些被锁的在位者拿不出来的。**

在一个 SOTA 被锁、用 AGPL、或只商用授权的品类里，做一个开放许可（Apache-2.0 / MIT）的模型。2026 年这块的可下手面大了一圈：TranscriptFormer、COMPASS、STATE、Closing-the-loop 多多少少都被锁或不够宽松，于是在好几个品类里都开出了宽松移植的机会——不再只是 Hibou 孤零零守着的病理那一块。

**代表作：Hibou**（HistAI 出品，在 Virchow2 / UNI2-h / CHIEF 一众被锁的病理 FM 里，那个 Apache-2.0 的）；COMPASS 式跨癌种 ICI 预测器的宽松移植；闭环 ISP 框架的干净 Apache-2.0 重实现；以及 TranscriptFormer 那一类跨物种 FM 的 HuggingFace 规范发布。

**底料自测：**你的发布工程讲得出一个什么样的故事？权重 + 代码 + 文档 + 一行在干净环境里真能跑起来的 `from_pretrained`，这是*最低线*。当下游团队把你的移植版当成事实上的 checkpoint 来用时，这条切入点才开始复利。

**技能：**一套能跑的模型配方（你不必是 SOTA）+ 认真的发布工程。算力是原配方需要多少就多少；护城河在于打磨。

## 切入点 6 — 互操作基础设施

**底料：一个你能守上十年的生态位置。**

让*别人*的工具反过来依赖你的软件——一种数据格式、一个工作流封装、一座跨语言的桥。沿用 v2，不变。

**代表作：SpatialData**（Stegle 组 + scverse）——2026 年空间组学 + 影像容器事实上的标准；**plyxp**（Mike Love）——把 dplyr 动词搬到 `SummarizedExperiment` 上；**bioc-to-galaxy**——LLM 辅助的 Bioconductor → Galaxy XML 翻译器；**scverse + scvi-tools + AnnData / MuData**；**生物 FM 的 HuggingFace 移植**；以及面向新模态的 **OME-NGFF / Zarr 扩展**。

**底料自测：**你在做社区 / 治理那摊事吗——回 GitHub issue、出席 scverse / OME 会议、写那些让别人真能用起来的文档？没有这些，你手里只有一堆库代码，不是基础设施。

**技能：**软件工程纪律 + 持续的社区工作。回报是结构性位置，不是引用数。

## 已退役——智能体 / 工具调用包装

**底料没了。**

v1 把它列为第 5 条，代表作是 X-Cell 和 MedAgentGym。半年过去，**CellForge**（面向单细胞扰动的多智能体神经架构搜索）、**VCHarness**（智能体 + 生物 FM，赢过专家设计的方案）、**BioLab**（带闭环 RAG 记忆的多智能体自主生命科学）和 **「Closing the loop」**（在 Perturb-seq 上迭代微调 Geneformer）已经把「智能体包 VC FM」这块从头到尾覆盖干净。**Biomni** 盖住了通用生物医学智能体那一面；**Google AI co-scientist** 盖住了假设锦标赛那一面；**TxGemma + Agentic-Tx** 盖住了「把治疗学 FM 当工具」那一面。v1 当年指望从智能体包装里拿到的那种组合式胜利，如今这些系统默认就给你产出了。

一个小团队当然还能发一个*特定*的智能体包装，只要它锚在一块大系统够不着的底料上——比如包住你那支有特权的队列，或者包住你那个病理 FM 跨癌种 ICI 闭环。但「智能体包 FM」这条*通用*切入点，关了。

## 叠加与底料复利

v2 那几种叠加套路（1+2、1+6、2+6）依然成立，而「底料即护城河」这套重排又添了一个有用的问题：**这次叠加，是在底料上复利，还是只在技术上复利？** 1+2 的叠加（同一个任务上「架构 + 适配器」）在生物学直觉这块底料上复利。3+5 的叠加（病人投影 + 宽松许可移植）在临床参考图谱这块底料上复利。4+6 的叠加（特权语料预训练 + 互操作格式）在数据准入这块底料上复利。这些才是能跑五年的长线项目。

*不*复利的叠加，是任何只锚在技术上的东西——一个 LoRA + 另一个 LoRA + 第三个 LoRA，给你三篇论文，零护城河。

## 与 v2 对账

v2 那次收口（替 ML+SWE 实验室收到第 1、2、6 条）是完好的、也是对的——*作为 v3 针对 v2 所描述的那种特定实验室画像的一个子选项*。v3 是更宽的分类法；v2 是其中一条特定的走法。要是你的实验室对得上 v2 那张画像（ML + SWE，没有特权数据，没有 FDA / 临床流水线），那 v2 的收口至今仍是更实用的读法。要是你的实验室有临床合作者准入或机构队列准入，那 v3 里的第 3、4、5 条就向你打开了。
