## 为什么会有 v3

2026-05-15 至 2026-05-18 之间发布的三轮审计，改变了什么才算得上一个可守的小团队动作：

- **v2 智能体闭环结论** 点名了四个已经发表的闭环 VC FM 系统（"Closing the loop"、VCHarness、CellForge、BioLab），并得出结论：闭环架构已不再新颖——剩下的是 *应用 + 纪律*。
- 闭环虚拟细胞的 **101 巡礼** 把领域级的心智模型说明白了：底料（rung-1 图谱、rung-2 扰动数据、纵向患者队列）是如今每一个有趣问题所系之处。
- **邻近形态 + n=1 随笔** 对三种后续项目形态（多细胞性 FM、抗 PD1 跨癌种、n=1 个性化）跑了同样的审计，又发现了三次同样的模式：闭环 / 模型要么已发表、要么已商品化，唯一可守的新颖性活在 *上游* 的底料决策里——湿实验图谱生成、评估纪律、纵向采样设计。

v2 小实验室随笔的三切入点收窄（架构、适配器、互操作）在它所及的范围内仍然正确——但它没说每条切入点守护的是哪种底料，而且即便闭环文献到现在已经把 v1 的智能体包装切入点商品化了，它仍让那条切入点留着。v3 把这两点都修正了。

## 这次重构——底料即护城河

到了 2026 年，闭环是大路货，骨干是大路货，架构大体上是大路货，包装也越来越是大路货。大 FM 实验室和资本充裕的智能体公司可以匹敌或击败其中任何一项。他们 *无法* 匹敌的是一道底料——某个特定的图谱、一支有特权的队列、一块临床参考面板、一个由社区治理的格式、一种别人没想过去编码进去的生物学直觉。下文每一条诚实的小实验室切入点，都由 *它守护的是哪种底料* 来定义，而不是由它用了哪种技术来定义。一个 LoRA 适配器不是护城河；*一个在你比任何人都更懂的下游任务上训练出来的 LoRA 适配器* 才是。

正是这次重构，解释了为什么 v2 智能体闭环随笔落在了"应用 + 纪律，而非方法"上，也解释了为什么邻近形态审计得出结论：**每种邻近形态里的护城河都活在模型的上游**。本文余下部分就是这条六切入点的推论。

## 按切入点划分的资源门槛

下面六条切入点在资源层面并不可同级相比。其中两条是货真价实的一个工程师一块 GPU 的动作；两条是披着小实验室外衣的机构级动作；还有一条对临床合作者的需要超过对 GPU 的需要。下表在各切入点小节给出底料 / 技能细节 *之前*，先点出每条切入点的资源闸门——先读它，再对照着读各小节。

| 切入点 | 算力门槛 | 数据 | 其他资源闸门 | 资源结论 |
|---|---|---|---|---|
| 1 — 架构归纳偏置 | 约 $5–20K 云算力 + 1 个 A100/A6000 节点 | CELLxGENE Census + sc-Arena / PerturBench 基准（公开） | 6–12 个月工程师时间 | **临界。** 预印本可达；SOTA 旗号不可达——为收敛到一个非 BERT 架构所需的迭代周期，现实里大约要 10× 的标称算力。 |
| 2 — 冻结骨干适配器 | $0–2K，本地硬件上一个 GPU-周 | Replogle、Tahoe-100M、GDSC、PRISM、CCLE——全部公开 | 一名工程师 × 6 个月 | **最干净的契合。** 唯一一条标称数字（"1–10 GPU-天"）诚实的切入点。 |
| 3 — 患者锚定投影 | 约 1 GPU-天，微不足道 | Sade-Feldman / Jerby-Arnon / Bassez / Bi 图谱是公开的；患者内验证样本不是 | **IRB + 临床合作者 + 活检 + DUA，往往还要一个医院隶属关系** | 算力很小；**临床合作者这个名额就是整道护城河**。没有它，这条切入点就是关着的。 |
| 4 — 特权语料持续预训练 | $20–100K（Geneformer-CLcancer 规模） | **按定义即排他** | 精选基础设施 + 机构隶属关系 | **机构级，而非小实验室级。** "特权语料"是机构访问权的另一个名字。 |
| 5 — 宽松许可重实现 | $5–20K（xVERSE 量级）至 $50–200K（Hibou / TranscriptFormer / COMPASS 量级） | 公开，但调和（harmonization）很贵 | 发布工程 + 持续维护 | **部分契合。** 只在小模型一端可行——大多数被门控的目标之所以被门控，正是因为复现它们很贵。 |
| 6 — 互操作基础设施 | 约 $0，一台笔记本就能跑 | 往往不需要 | 多年持续的工程师时间 + 社区 / 治理工作 | **与 W2 并列最干净的契合。** 瓶颈是年头，不是钱。 |

对照这张表，只有 **切入点 2 和切入点 6** 在资源层面是毫不含糊的小实验室契合。**切入点 1** 凭一个 GPU 节点和一笔不大的云预算就过线。**切入点 3** 对临床合作者的需要超过对算力的需要——把它作为小实验室切入点的 v3 框定，只有在那个名额被填上的条件下才诚实。**切入点 4 和 5** 是披着小实验室外衣的机构级动作，尽管下面各切入点的散文把它们写得像是与切入点 1、2、6 同级可比。底料即护城河的重构正是 *为此命名* 的东西——护城河在模型的上游，而上游资源（数据、临床访问权、持续工程）的大部分，并不是一个小实验室用一块 GPU 加一年时间能买到的。

## 切入点 1 — 架构归纳偏置

**底料：你的生物学直觉。**

一个对生物学专门正确的新架构组件或完整骨干——而非从 NLP 重新挪用的 transformer。大 FM 实验室困在 BERT 的路径依赖里；你的防守是你愿意在架构层面编码进去的某一块生物学。

**范例**（大体上沿用自 v2）：**xVERSE**（Jiang & Xie, bioRxiv 2026.04）——转录组原生的非 LM 架构，相对 scGPT / Geneformer / scFoundation / UCE 有 11–34% 的提升；**TxPert**（Wenkel et al., *Nat Biotech* 2026）——把多知识图谱先验注入扰动 transformer；**ESM-3 的 SE(3) 不变 token**；**AlphaGenome 的轨迹预测目标**；用于基因组底料的状态空间骨干（Hyena / Mamba / Evo2）。

**底料检验：** 问问你的架构编码了哪一种对称性、先验或生物学结构，是普通 transformer 所没有的。如果你能用一句话说出来，这就是一条真切入点。如果说不出来，你不过又造了一个 BERT。

**技能：** PyTorch/JAX + 有主张的生物学。算力是货真价实地不大（个位数 GPU-周）。

## 切入点 2 — 冻结骨干适配器

**底料：一个你比任何人都更懂的下游任务。**

一个 LoRA / Houlsby 适配器 / 专用头部（占骨干参数的 0.1–5%），用来修正预训练 FM 的某一个特定失效模式。2025 年的线性基线清算让这成了这个领域里杠杆最高的动作。

**范例：** **sc-FM Perturbation Adapter**（ICLR 2026）——一个 <1% 的、以药物为条件的适配器，击败了线性可加基线；**PertAdapt**（bioRxiv 2025）——带基因级结构、对条件敏感的适配器；**scDCA**（ICLR 2026）——冻结 sc-FM 上的药物条件适配器；细胞类型特异的分类头部；sc-FM 与 spatial-FM 嵌入空间之间的跨模态适配器；用于病理 FM（CONCH, TITAN, CHIEF）的 attention-MIL 聚合器。

**底料检验：** 说出那个下游任务，并解释 *为什么你的任务知识胜过下一个团队的超参搜索*。如果答案是"我只是把头部调得更好了"，那是一篇论文，不是一道护城河。

**技能：** PyTorch + 深厚的任务知识。算力是每个实验 1–10 GPU-天。

## 切入点 3 — 患者锚定投影（新增）

**底料：某个特定的临床参考图谱 + 纵向采样纪律。**

这条切入点在 v1 或 v2 里都不存在。它在 [邻近形态随笔](adjacent-project-shapes-and-n-of-1.md) 里的 n=1 微调审计中被明确命名出来，而且它是一个货真价实的小团队动作：取一份已发表的应答者图谱（Sade-Feldman 黑色素瘤、Jerby-Arnon、Bassez 乳腺、Bi RCC），用 **scArches**（[Lotfollahi et al., *Nat Biotech* 2021](https://www.nature.com/articles/s41587-021-01001-7)）把新的患者数据投影进去，接上一个冻结 FM 的特征提取器 + 在队列上训练的逻辑回归 / GBM 头部，并把结果作为一个 HuggingFace Space 或一个临床研究工具发布出去。

让它成为一条切入点的原因：每篇闭环 VC 论文都假设你能微调一个骨干，而每一个 n=1 个性化的主张都会撞上灾难性遗忘那堵墙。scArches 式的投影绕开了这两者——冻结骨干、冻结 FM、在队列上训练的头部、*逐患者推理*。跨患者时"模型"不变；变的是每个患者在图谱中走出的那条投影轨迹。这是 2026 年文献里唯一诚实的 n=1 动作，而几乎没有人把它做成一个 *产品化的* 小团队成品。

**范例：** **scArches** 本身及其后裔；**Tempus IPS** 与 **PurIST 胰腺** 模式（在队列上训练、逐患者查询的头部）；患者特异的 **Virchow2** + 在 H&E 上用于 ICI 应答的 attention-MIL 聚合器，在留一癌种（leave-one-cancer-out）评估下（这是 [邻近形态随笔](adjacent-project-shapes-and-n-of-1.md) 里命名的那个可守利基）。

**底料检验：** 在写代码 *之前* 就说出参考图谱 + 留出的评估队列。如果它们不存在或不可访问，这条切入点对你就不开放。

**技能：** scArches/scvi-tools 的熟练度 + 一个能提供患者内纵向样本供验证的临床合作者。算力是每次投影一个 GPU-天。

## 切入点 4 — 特权语料持续预训练

**底料：你能拿到、别人拿不到的数据。**

取一个开放权重的 FM，在一份只有你能拼起来的、严格精选的领域语料上继续预训练，然后发布新的检查点。在 v1 里是切入点 3。

**范例：** **Geneformer V2-104M_CLcancer**——通用 Geneformer 在 14M 精选癌症细胞上持续预训练得到；尽管小了 3×，却在大多数癌症下游任务上匹敌或击败 316M 的通用模型。

**底料检验，加硬版：** 是什么让你的语料 *排他*？如果答案是"我们从 CELLxGENE Census 拉的"，那这条切入点不属于你——大 FM 实验室早已在 Census 上预训练过。这条切入点只有在你拥有一份特权语料时才开放（某机构的罕见病队列、某个组织特异的档案、一份别人碰不到的临床试验收藏）。

**技能：** 数据工程（语料精选是大部分工作）+ 足够的 HuggingFace 熟练度以干净地继续预训练。算力依语料规模为 10–100 GPU-天。

## 切入点 5 — 宽松许可重实现

**底料：被门控的在位者无法匹敌的发布工程 + 许可清晰度。**

在某个 SOTA 是门控、AGPL 或仅限商业的类别里，做一个开放许可（Apache-2.0 / MIT）的模型。2026 年这里的可及面大幅扩大了：TranscriptFormer、COMPASS、STATE、Closing-the-loop 都以某种形式被门控或非宽松，从而在多个类别里开辟了宽松移植的机会——不只是 Hibou 曾孤身一人的病理领域。

**范例：** **Hibou**（HistAI，在 Virchow2 / UNI2-h / CHIEF 被门控的格局里的 Apache-2.0 病理 FM）；COMPASS 式跨癌种 ICI 预测器的宽松移植；闭环 ISP 框架的干净 Apache-2.0 重实现；TranscriptFormer 量级的跨物种 FM 的 HuggingFace 规范化发布。

**底料检验：** 你的发布工程故事是什么？权重 + 代码 + 文档 + 一行在全新环境里真能跑通的 `from_pretrained` 是 *底线*。当下游团队把你的移植采纳为事实上的检查点时，这条切入点就会复利增长。

**技能：** 一份可用的模型配方（你不必是 SOTA）+ 认真的发布工程。算力是原配方所需的任何量；护城河在于打磨。

## 切入点 6 — 互操作基础设施

**底料：一个你能守上十年的技术栈里的社区位置。**

让 *别人的* 工具去依赖的软件——一个数据格式、一个工作流包装器、一座跨语言的桥。沿用自 v2。

**范例：** **SpatialData**（Stegle 组 + scverse）——2026 年空间组学 + 成像容器的事实标准；**plyxp**（Mike Love）——在 `SummarizedExperiment` 上的 dplyr 动词；**bioc-to-galaxy**——LLM 辅助的 Bioconductor → Galaxy XML 翻译器；**scverse + scvi-tools + AnnData / MuData**；**生物学 FM 的 HuggingFace 移植**；面向新模态的 **OME-NGFF / Zarr 扩展**。

**底料检验：** 你在做社区 / 治理工作吗——回应 GitHub issue、参加 scverse / OME 会议、写出能让采纳真正可行的文档？没有这些，你拥有的是库代码，不是基础设施。

**技能：** 软件工程纪律 + 持续的社区工作。回报是结构性位置，不是引用数。

## 已退役 — 智能体 / 工具调用包装器

**底料没了。**

v1 把这条列为切入点 5，范例是 X-Cell 和 MedAgentGym。六个月后，**CellForge**（用于 sc 扰动的多智能体神经架构搜索）、**VCHarness**（智能体 + 生物学 FM 击败专家设计的方法）、**BioLab**（带闭环 RAG 记忆的多智能体自主生命科学），以及 **"Closing the loop"**（在 Perturb-seq 上对 Geneformer 迭代微调）端到端地覆盖了 VC FM 之上的智能体包装空间。**Biomni** 覆盖了通用生物医学智能体面；**Google AI co-scientist** 覆盖了假设锦标赛面；**TxGemma + Agentic-Tx** 覆盖了把治疗 FM 当工具用的那一面。v1 期望从智能体包装里得到的那些组合性收益，如今这些系统默认就能产出。

一个小团队仍然可以发布一个 *特定的* 智能体包装器，锚定在大系统够不到的某种底料上——例如，一个围绕你那支特权队列、或围绕你那条病理 FM 跨癌种 ICI 闭环的智能体包装器。但 *通用的* "FM 之上的智能体"切入点已经关上了。

## 堆叠与底料复利

v2 的堆叠模式（1+2、1+6、2+6）依然成立，而底料即护城河的重构添了一个有用的新问题：**这个堆叠是在底料上复利，还是只在技术上复利？** 一个 1+2 堆叠（同一任务上的架构 + 适配器）在生物学直觉底料上复利。一个 3+5 堆叠（患者投影 + 宽松许可移植）在临床参考图谱底料上复利。一个 4+6 堆叠（特权语料预训练 + 互操作格式）在数据访问底料上复利。这些才是经得起 5 年的持久项目。

*不* 复利的堆叠，是任何只锚定在技术上的东西——一个 LoRA + 另一个 LoRA + 第三个 LoRA，给你三篇论文，零护城河。

## 与 v2 的对账

v2 的三切入点收窄（为 ML+SWE 实验室的 1、2、6）完好且正确，*作为针对 v2 所描述的那个特定实验室画像、对 v3 所做的一次子选择*。v3 是更宽的分类法；v2 是其中一条特定的遍历路径。如果你的实验室符合 v2 画像（ML + SWE、没有特权数据、没有 FDA / 临床流水线），v2 的收窄仍是务实的读法。如果你的实验室有临床合作者访问权或机构队列访问权，v3 里的切入点 3、4、5 就向你开放了。
