> *受众：一个偏 ML 的博士后或初级 PI，1-5 人，没有 10 亿美元的算力预算，没有 800 家医院的数据合作，但真心希望在虚拟细胞 / 基础模型空间里 **出货点东西**，而不是再写一篇批评论文。本文是[基础模型全景现状](foundation-models-state-of-play.md)（已经构建了什么）和[如何造一个生物学 FM](how-to-build-a-biological-fm.md)（大 FM 团队怎么造）的配套。*

2025-2026 年的标题故事——Lilly + NVIDIA 的 10 亿美元 Vera Rubin 交易、Owkin 的 800 家医院网络出货进入 Claude for Healthcare、Genentech 的图谱级努力——看起来这个领域属于少数几个资本雄厚的团队。那在 *checkpoint* 层面是真的。但在 *artifact* 层面就不真了，在那一层，1-5 人的团队在 2026 年出货了真东西。本文先勾画出你跨不过的护城河，然后把小团队真正出货过的六个切入点形态分类——每个都配一个语料里的范例，外加一行"你需要什么技能"的标签。

## 护城河——你竞争不了的东西

一个小实验室在 2026 年造不出的三样东西：

1. **一个全新的、从头开始的通用 sc-FM。** 单是数据协调（100M+ 细胞、数百个供体、跨平台跨物种的基因符号统一）就是一个多年的工程项目，而这还在你写第一行模型代码之前。算力随后再加上 $5-50M。
2. **一个全新的通用病理 FM。** Mahmood 实验室的 UNI 语料是来自多机构合作的 >100M 张 H&E 瓦片；Owkin 横跨 800 家医院训练。靠一份博士后的薪水你组装不出一个可比的训练语料。
3. **一个面向消费者的分发位置。** 当 Anthropic 把一个 FM 嵌进 Claude for Healthcare，那是一道单凭模型无法击败的分发护城河。

接受这三点，能让你自由地去挑一个不与它们正面交锋的切入点。

## 切入点 1——归纳偏置模块

**构建：** 一个对生物学专门正确的新架构组件——而不是从 NLP 移植来的 transformer。**范例：** [xVERSE](https://liudengzhang.github.io/conference-vaults/conferences/iclr-2026/tools/)（Jiang & Xie，bioRxiv 2026.04）——一个转录组原生的非 LM 架构，在表征、批次校正和空间填补上以 **11-34%** 击败了每一个主要的 sc-FM。另见 [TxPert](https://doi.org/10.1038/s41587-026-03113-4)（Wenkel 等），它为转录组扰动预测赋予了一个多知识图谱先验。这个切入点：大 FM 实验室被困在 BERT 克隆的路径依赖里；你没有。**技能：** ML 建模 + 严肃的生物学素养（你必须 *知道* 哪个归纳偏置才是对的那个）。

## 切入点 2——冻结骨干上的小适配器

**构建：** 一个 LoRA / 适配器 / 专用头（<1-5% 的骨干参数），用以修复预训练 FM 做不到的一个特定任务。**范例：** [sc-FM Perturbation Adapter](https://liudengzhang.github.io/conference-vaults/conferences/iclr-2026/tools/sc-fm-perturbation-adapter/)——一个在冻结 sc-FM 骨干上的 <1% 药物条件化适配器，在零样本扰动预测上击败了线性加性基线。这个切入点：当骨干的预训练目标与下游任务不匹配时（这对几乎每个 sc-FM 在扰动上都成立），一个小的专用头就能把它救回来。算力是单 GPU 一周。**技能：** PyTorch 熟练度 + 一个你比任何人都更懂的特定下游任务。

## 切入点 3——领域特定的持续预训练

**构建：** 取一个现有的开放权重 FM，在一个紧密精选的领域语料上继续预训练，然后出货这个新 checkpoint。**范例：** **Geneformer V2-104M_CLcancer**——把通用 Geneformer 在 14M 精选癌症细胞上继续预训练。这个癌症领域的 checkpoint 在大多数癌症下游任务上追平或击败了 316M 的通用模型，尽管它 **小了 3 倍**。这个切入点：大 FM 实验室为广度而训练；你为一种疾病、一种细胞类型、一种组织而训练，并在深度上取胜。**技能：** 数据工程（语料精选是大部分工作）+ 足够的 HuggingFace 熟练度以干净地继续预训练。

## 切入点 4——宽松许可的替代品

**构建：** 在一个 SOTA 被门控、AGPL 或仅商用的类别里，做一个公开许可（Apache-2.0 / MIT）的模型。**范例：** **Hibou**（HistAI，*不是* Owkin）——目前在一个本来由门控权重主导的类别里（Virchow2 / UNI2-h 需要许可申请；CHIEF 是 AGPL-3.0），唯一一个 Apache-2.0 的顶级病理 FM。Hibou 由一个极小的团队出货，并因为许可摩擦在学术和商业场景里都是一笔真实的采用税而被广泛采用。这个切入点：一个略落后于 SOTA 但许可干净的模型，往往在集成进别人的流水线时取胜。**技能：** 一个能用的模型配方（你不必是 SOTA）+ 对发布工程的认真投入（真正能用的权重 + 代码 + 文档）。

## 切入点 5——智能体 / 工具使用封装器

**构建：** 一个 LLM 驱动的智能体，编排现有的 FM 而非与它们竞争——把 scGPT + Geneformer + UNI + AlphaGenome 组合进一个任何单一 FM 都做不到的工作流。**范例：** **X-Cell**（Bo Wang 实验室）——一个围绕 sc-FM 栈的智能体封装器，端到端地编排分析。[MedAgentGym](https://liudengzhang.github.io/conference-vaults/conferences/iclr-2026/tools/medagentgym/) 提供了这个类别所需的评估面（72k 任务，沙箱化代码执行）。这个切入点：在许多生物学任务上，下一个增益来自现有 FM 的 *组合*，而非来自造一个更好的。Anthropic 的 MCP 让封装变得可处理。**技能：** LLM 编排（智能体、工具使用协议、评估框架）+ 生物学工作流知识。

## 切入点 6——互操作基础设施

**构建：** *别人的* 工具所依赖的软件——一种数据格式、一个工作流封装、一座跨语言的桥。**范例：** **SpatialData**（Zarr / OME-NGFF 跨语言框架，如今每个现代空间组学工具都以它为目标）、**plyxp**（在 SummarizedExperiment 上的 dplyr 动词——Mike Love 的桥）、**bioc-to-galaxy**（LLM 辅助的 Bioconductor → Galaxy XML 翻译器）。这个切入点：FM 需要底料、胶水和管道，而造 FM 的人不会把管道造好。基础设施会复利——一个一年的格式被采用，一个五年的格式成为必然。**技能：** 软件工程纪律 + 社区 / 治理工作（这是当一个好的开源公民最能获得回报的切入点）。

## 该跳过什么

三个对 2026 年小团队诱人却贫瘠的模式：

- **"又一个 sc-FM"**，在同样的 CELLxGENE Census 上、带一个新的架构小花样训练。清算文献表明，来自一个新骨干的增益至多温和，而十份带小改动的 scGPT 副本没有带来任何实践改变。
- **"又一篇线性基线批评论文。"** 清算已经打赢了；下一篇论文的回报递减。用户特别要求的是构建形态的切入点，而这个类别是分析形态的。
- **"独立基准"** 作为一篇论文而非作为软件发布。sc-Arena 和 PerturBench 之所以获得牵引力，是因为它们作为可复用的基础设施出货。一个只存在于 PDF 里的基准是一份出版物，不是一件 artifact。

## 挑你的切入点

三个决策问题，按顺序：

1. **我愿意维护一件 artifact 3 年以上吗？** 切入点 4 和 6（Hibou 式的模型、互操作基础设施）随时间复利，但需要持续的工程。切入点 1 和 2（架构、适配器）可以作为一篇论文加代码一次性出货然后停下。
2. **我的数据访问如何？** 如果你有一个特权语料（一种罕见病、一个机构的空间组学档案），切入点 3（持续预训练）是杠杆最高的招法。如果你只有公开语料，就倾向切入点 1、2、5。
3. **我是擅长架构创造、软件创造，还是组合创造？** 切入点 1 要的是新 ML，切入点 6 要的是新基础设施，切入点 5 要的是新编排。它们奖励不同的肌肉。

诚实的总结：每一个 2025-2026 年的小团队成功，要么（a）为生物学做了某件大 FM 实验室因太 BERT 上头而不愿尝试的、专门正确的事，要么（b）出货了 *使用* 大 FM checkpoint 而非与之竞争的基础设施 / 适配器 / 智能体。这两个切入点在 2026 年都敞开着。
