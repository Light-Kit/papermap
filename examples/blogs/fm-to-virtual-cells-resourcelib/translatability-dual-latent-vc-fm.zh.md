## 问题是什么

拿一个虚拟细胞 FM，在同一个骨干上微调两次——一次在**细胞系药物响应**上（GDSC + Tahoe-100M），一次在**病人药物响应**上（TCGA 配对队列、BeatAML、PharmacoDB）。你就得到两个隐空间。把它们摆到一起比——不只比谁预测得准，更要比机制：药物响应的哪些维度能跨系统保留下来，哪些一过去就塌了。最后这一句才是真正值钱的东西：一个能解释**翻译为什么会失败**的模型，而不是一个光会预测响应的模型。还有，适配器得你自己造——它是为了那场**比较**而设计的，不是为了响应头。

## 一片挤了七年的前人工作

「细胞系 → 病人」的药物响应迁移问题，比这个提法听上去更老、人也更多。小编码器那个时代从 2019 跨到 2024：

- **PRECISE**（Mourragui, *Bioinformatics* 2019, [doi:10.1093/bioinformatics/btz372](https://doi.org/10.1093/bioinformatics/btz372)）——用对比 PCA 求线性共识子空间。这一脉的开山。
- **MOLI**（Sharifi-Noghabi, *Bioinformatics* 2019）——后期整合的 triplet-margin 嵌入。
- **AITL**（Sharifi-Noghabi 2020）——输入端 + 输出端的对抗式域适应。
- **TRANSACT**（Mourragui, *PNAS* 2021, [doi:10.1073/pnas.2106682118](https://doi.org/10.1073/pnas.2106682118)）——核 PCA 的非线性扩展，把共享的转录组分量浮出来。
- **TUGDA**（Peres da Silva, *Bioinformatics* 2021）——任务不确定性引导的 DA；把负迁移压低了 94%。
- **CODE-AE**（He/Xie, *Nat Mach Intell* 2022, [doi:10.1038/s42256-022-00541-0](https://doi.org/10.1038/s42256-022-00541-0)）——一个自编码器，显式地把隐空间正交分解成 `共享 + 系统特有`。**这才是用户那套想法直系的架构祖宗。**
- **scDEAL**（Chen, *Nat Commun* 2022）——用 DaNN + MMD 做 bulk-到-sc 的域适应。
- **scAdaDrug**（[arXiv:2403.05260](https://arxiv.org/abs/2403.05260), 2024）——重要性感知的多源 DA，带一个逐隐特征的权重生成器。
- **PERCEPTION**（Sinha/Ruppin, *Nat Cancer* 2024, [doi:10.1038/s43018-024-00756-7](https://doi.org/10.1038/s43018-024-00756-7)）——临床试验验证的脚印最硬的一个。**关键一点：它不是 VC FM**——是跑在配对 bulk + sc 上的 DNN。

接着 FM 时代开场，2024-2026：

- **CRISP**（*Nat Comput Sci* 2025）——第一个用 sc-FM 当骨干的迁移；做的是细胞类型 → 细胞类型，而不是细胞系 ↔ 病人。
- **scDrugMap**（*Nat Commun* 2025）——把 8 个 sc-FM 拉到响应分类上做基准；汇总下 scFoundation 最强，微调后 UCE 最佳。
- **UniCure**（[bioRxiv 2025.06.14.658531](https://www.biorxiv.org/content/10.1101/2025.06.14.658531v1)）——**离得最近的那个在世对手，独一份**。真用了 FM 骨干（UCE + Uni-Mol），在 180 万条扰动谱上预训练，再在病人来源的类肿瘤簇上微调。但它只做了**一次**病人微调，没有配对的细胞系微调，也没有显式的 Δ-隐空间分析。
- **TRANSPIRE-DRP**（*J Transl Med* 2025）——PDX → 病人的 AE + 对抗式 DA。

**没有谁把「VC FM + 配对的双微调 + Δ-隐空间比较 + 逐基因 τ 输出」做成同一个系统。** UniCure 才半年大，又在配对微调这一步上缺席。CODE-AE 是小编码器尺度上的架构祖宗。其余的要么没用 FM 骨干，要么没做双微调，要么没有基因层面的读出。

## 那个空着的逐基因 τ_g 格子

先把双隐空间分解放一放，单看**模型输出的是什么**。把「细胞系→病人」迁移这一摊文献、逐基因域适应这一摊文献、药理学里那些可迁移性评分量表、还有 2024-2026 年的 FM 时代工作都扫过一遍之后——**没有一篇论文，会输出一个学出来的、逐基因的标量 τ_g ∈ [0,1]，去说「基因 g 在细胞系里的响应，能以置信度 τ 迁移到病人响应上」。**

最接近的那几样现成品，各有各的缺口：

- **scAdaDrug**——逐隐特征、逐样本的权重，没有稳定的生物学身份（你读不出「EGFR 能迁移、KRAS 不能」这种话）。
- **TUGDA**——逐任务（逐药）的不确定性，不是逐基因。
- **CODE-AE**——隐空间里逐样本的对齐，没有分到基因这一级的读出。
- **TRANSACT**——因子 / 分量级的载荷，不是逐基因。
- **PDXGEM**（*BMC Med Genomics* 2020）——一个二值的一致性过滤器，不是一个学出来的连续 τ。
- **Sakellaropoulos**（*Front Genet* 2023）——KS 检验过滤器 + 排序面板。是离散的，不是端到端学出来的。
- **Wendler & Wehling 可迁移性评分**（*J Transl Med* 2017, [PMC5670516](https://pmc.ncbi.nlm.nih.gov/articles/PMC5670516/)）——药物项目层面的一套人工打分量表。同一个词，完全不同的对象。
- **pRRophetic / oncoPredict**——用一个多基因岭回归模型，去推算病人的 IC50。那些系数是样本预测的权重，不是迁移置信度的分数。

τ_g 这个提法是真正空着的。三种可以预想到的审稿人反对，附上诚实的回应：

1. *「pRRophetic 精神上不就在做这件事。」* 不。pRRophetic 输出的是样本预测；它的系数不是学出来的逐基因迁移置信度，生物标志物的发现是事后通过 IDWAS 补上的。
2. *「逐样本重要性加权的 DA，功能上不就是一回事。」* 不。逐样本的权重把*「这个病人不寻常」*和*「这个基因不可翻译」*混在了一起。一个 τ_g 头是显式地对样本做边缘化、在基因这条轴上可辨识的——是个不一样的统计对象。
3. *「可迁移性是逐药的，不是逐基因的。」* 部分有理。诚实的因子化应该是 τ_{g,d}。但「细胞系→肿瘤」的那几种失败模式（微环境驱动的基因、细胞系里没有的免疫基因、细胞系里被吹大的细胞周期通路）大体上是*基因内禀的*，会跨药物类别反复出现——见 Yu et al., *Sci Adv* 2020, [PMC7458440](https://pmc.ncbi.nlm.nih.gov/articles/PMC7458440/)。所以一个边缘的 τ_g 仍然定义得清楚、也仍然有用。

## 自己造的那个适配器——损失优先，架构从简

VC FM 之上的适配器形态，已经发出来十二种了：scDCA（由药物嵌入调制的 FiLM-on-bias）、[PertAdapt](https://www.biorxiv.org/content/10.1101/2025.11.21.689655)（基因相似度掩码的注意力 + 自适应的扰动敏感基因损失）、scPEFT（LoRA / Adapter / Prefix）、FUSED（scGPT 和 MolFormer 之间的多头交叉注意力）、scGPT-DRP-GNN、XTransferCDR、scGenePT（在基因 token 处注入文本）、Bio-DTA（动态 token 重写）、PertDiT（扩散-transformer 的交叉注意力）、端到端的 STATE，再加上 CellForge 和 VCHarness 这两个智能体式生成器。**纯架构上的新意，基本已经关门了。** 再来一种 FiLM 的新口味，读起来就像一次跟风迭代。

而**在一个冻结的 VC FM 上没人试过**的，是这些：

1. **双头共干适配器**，在瓶颈上挂一个 **CKA / OT / Procrustes 对齐损失**。细胞系头和病人头共享一个主干；瓶颈处的相似度，告诉你哪些维度熬过了翻译。
2. **逐基因 τ_g 头**，作为跑在 FM 特征上的 DR-learner 或 EP-learner。
3. **域条件 FiLM**，由一个 `{细胞系, PDX, 病人}` 的 token 来驱动逐层的 γ/β。
4. **辅助的可迁移性标量**，逐药地预测「细胞系→病人」的可迁移程度。
5. **配对匹配的（细胞系, 肿瘤）对比损失**，跑在一个 VC FM 骨干上。

**2026 年的「造你自己的适配器」，意思就是损失优先、架构从简。** 新意得活在*目标函数*和*比较机器*里，而不是在瓶颈那点算术上。最强的一套堆法，一口气命中四个空格子：**冻结的 STATE 骨干 + 共干双头适配器（细胞系头 + 病人头）+ 瓶颈上的 CKA/OT 对齐损失 + 逐基因 τ_g 的 DR-learner 头 + 逐药的辅助可迁移性标量。** 瓶颈本身可以就是普通的 LoRA，或者 scDCA 式的 FiLM-on-bias——瓶颈那点算术，不是你拼的地方。

## 数据怎么走

底料是有的，但**配对的**底料比宣传里说的要窄：

- **GDSC2 × TCGA**——主力；约 30-60 种药有可用的重叠（顺铂、紫杉醇、吉西他滨、5-FU、索拉非尼、厄洛替尼、他莫昔芬、替莫唑胺，是覆盖得比较足的几个）。
- **CCLE / PRISM × TCGA**——细胞系那侧有 4,518 种化合物 × 578 个细胞系。
- **BeatAML 2.0**（*Cancer Cell* 2022）——805 名 AML 患者，带 145 种抑制剂的离体响应 + RNA-seq。**最干净的单适应证试验床**：venetoclax 有 367 个配对样本，跟 CCLE-AML（GDSC 上约 40 个细胞系）配。
- **NIBR-PDXE**（Gao, *Nat Med* 2015）——约 1,000 个 PDX × 60 种处理。**第三个域的桥**，给「细胞系 → PDX → 病人」这条扩展用。
- **Sade-Feldman 黑色素瘤 anti-PD1**（*Cell* 2018）——48 名患者。**那个阴性对照的证伪器**：anti-PD1 没有细胞系响应，因为细胞系没有免疫系统。τ_g 在 anti-PD1 这条轴上必须 ≈ 0。要是你的头在这些基因上回出了 τ > 0，那就是过拟合，项目就死了。这是一个又锐又单一的测试。
- **Tahoe-100M**（Vevo + Arc, 2025）——1 亿细胞 × 1,100 种药 × 50 个细胞系，对细胞系那侧来说是过定的。

## 不能踩的雷 + 该盯的人

还是 v2 那个套路。诚实的雷区是：

1. **要是你的「VC FM + 自造适配器」在一个封存的病人队列上打不过 CODE-AE，那这个 FM 就是个摆设。** CODE-AE 是小编码器尺度上的架构祖宗。把它当成一个*对照条件*来跑，而不是只引一下。如果打平甚至被它赢了，那这个 FM 骨干就没赚回它烧的算力。
2. **要是 τ_g 在 Sade-Feldman anti-PD1 上没被证伪，那这个头就是在过拟合。** 训练之前就把这个测试钉死。它是文献给得出的、最便宜的单一证伪装置。
3. **要是 UniCure，或者某个被移植到双域设置里的 CRISP 变体，在封存队列上跟你那套定制系统打平了，那你这个方法的贡献就是零。** UniCure 才半年大，又走在一条相似的轨迹上；把它当成在世的那条标杆线对待。

三个该盯的人：**Lodewyk Wessels**（NKI）和 **Soufiane Mourragui**（Sanger）——PRECISE → TRANSACT 这一脉，以及 2026 年那批样本高效适配的工作；**Eytan Ruppin**（NCI）和 **Sanju Sinha**（Sanford Burnham）——PERCEPTION，以及最硬的那份临床试验验证脚印；**Lei Xie**（Hunter / CUNY）——CODE-AE，双隐空间分解的直系祖宗。值得一提的还有：**UniCure 团队**（在世竞争；开工前先跟他们聊聊）、Hossein Sharifi-Noghabi（Roche）、Qin Ma（OSU；scDEAL / scDrugMap），以及——如果三域扩展真要上桌的话——那个 **NIBR-PDXE** 团队。

结论：还是 v2/v3 那个套路，只是往上挪了一层。双隐空间分解已经被发表了。FM 骨干的版本，有一个在世对手。**逐基因 τ_g 头，才是那个真正空着的格子**——把它定为承重的主张，围着「让 τ_g 可辨识」去设计适配器和损失，并且在动其他任何东西之前，先在 Sade-Feldman 上把它证伪。架构上的搭法不是你的护城河；比较机器才是。
