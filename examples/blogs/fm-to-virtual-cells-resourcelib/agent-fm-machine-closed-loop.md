> *The closed-loop sibling to the [reachability program](cell-state-reachability-as-viability-theory.md) and the [diffusion / flow-matching backbone](diffusion-and-flow-matching-for-virtual-cells.md) that proposes candidates inside it. Where [agentic meets foundation](agentic-meets-foundation.md) maps the software-only agent×FM patterns and the [closed-loop 101 tour](closed-loop-virtual-cells-101.md) walks the field history, this essay adds the third layer — the physical machine — and asks what the loop looks like when one corner of it is a robot. Prioritization runs on the [surprise / uncertainty signal](surprise-and-uncertainty-in-cell-fms.md); the project verdict lives in [agent-loop-for-drug-response v2](agent-loop-for-drug-response-v2.md).*

## Why three layers, not two

An LLM agent is a policy: it decides what to do next, but it has no world model worth trusting and no hands. A foundation model is a world model: it carries a learned density over cell state, but it has no ground truth — it only ever saw realized data — and it also has no hands. A self-driving lab has both hands and ground truth: it can run the experiment and return the real answer, but it has no idea *which* experiment is worth running. Each layer's strength is exactly another's blind spot, which is why the interesting systems compose all three rather than any two. The agent is the policy, the foundation model is the surrogate simulator, and the machine is the environment and the oracle. Framed that way the architecture is not new — it is model-based reinforcement learning, or equivalently active learning with a learned surrogate and a real oracle in the loop. What is new is that for the first time all three corners exist as deployable systems at once, and the corner everyone underweights is the third one: the physical machine that turns a prediction into a fact.

## The economic logic: arbitraging cheap-wrong against expensive-right

The whole loop runs on a single asymmetry. The foundation model gives you a *cheap* answer that is *probably wrong* — a forward pass costs milliseconds and returns a density, a likelihood, a predicted perturbation response. The machine gives you an *expensive* answer that is *right* — a real perturb-seq run or a reprogramming attempt costs dollars and weeks and returns ground truth. The agent's entire job is to arbitrage that gap: spend the cheap surrogate freely to decide which of the expensive experiments is worth doing. This is classical Bayesian experimental design — pick the experiment that maximizes expected information gain per unit cost ([Foster et al., ICML 2021](https://arxiv.org/abs/2103.02438)) — and its cousins Bayesian optimization ([Frazier, 2018 tutorial](https://arxiv.org/abs/1807.02811)) and active learning ([Settles, 2009 survey](https://burrsettles.com/pub/settles.activelearning.pdf)). The acquisition function is where the FM earns its keep: it proposes candidates and, crucially, an *uncertainty* on each one (the [surprise / uncertainty signal](surprise-and-uncertainty-in-cell-fms.md)), so the agent can target the points where a real measurement would teach the model the most rather than the points it is already confident about. When candidates must be both high-value and *diverse* — you do not want a batch of near-identical experiments — generative samplers like GFlowNets ([Bengio et al., 2021](https://arxiv.org/abs/2106.04399)) replace the single-best acquisition with a portfolio. The result of each round of real experiments updates the FM (or, more cheaply, an adapter on a frozen FM), the density sharpens where it mattered, and the loop turns again.

## The machines that already close the loop

The physical layer is not speculative; it is just further along in chemistry and materials than in cell biology. The original **Robot Scientist Adam** ([King et al., *Science* 2009](https://www.science.org/doi/10.1126/science.1165620)) autonomously formed hypotheses about yeast gene function, designed experiments, ran them on its own hardware, and revised — the first end-to-end closed loop with no human in the inner cycle. The **mobile robotic chemist** ([Burger et al., *Nature* 2020](https://www.nature.com/articles/s41586-020-2442-2)) ran 688 experiments over eight days to optimize a photocatalyst, a free-roaming arm rather than a fixed liquid handler. **A-Lab** ([Szymanski et al., *Nature* 2023](https://www.nature.com/articles/s41586-023-06734-w)) synthesized novel inorganic materials autonomously, ML-guided from the Materials Project convex hull. **Coscientist** ([Boiko et al., *Nature* 2023](https://www.nature.com/articles/s41586-023-06792-0)) put a GPT-4 agent in charge of a cloud lab, planning and executing real reactions from a natural-language goal — the cleanest agent-plus-machine exemplar to date — and **ChemCrow** ([M. Bran et al., *Nat Mach Intell* 2024](https://www.nature.com/articles/s42256-024-00832-8)) wired an LLM to a toolbox that can drive synthesis. The broader pattern has a name now, the self-driving lab ([Abolhasani & Kumacheva, *Nat Synth* 2023](https://www.nature.com/articles/s44160-022-00231-0)). On the biology side the hands exist — programmable cloud labs (Emerald, Strateos), biofoundries (Ginkgo), and automated phenomics at scale (Recursion's Cell Painting pipelines, [Bray et al., *Nat Protocols* 2016](https://www.nature.com/articles/nprot.2016.105)) — but the *autonomous* inner loop on single-cell readouts is still early.

## Where the foundation model actually sits in the loop

For virtual cells the FM is the surrogate model inside that experimental-design loop, and the [reachability program](cell-state-reachability-as-viability-theory.md) shows exactly where each piece plugs in. The generative backbone — a [diffusion or flow-matching model](diffusion-and-flow-matching-for-virtual-cells.md) — proposes candidate engineerable states or reprogramming cocktails: a Schrödinger-bridge path from a fibroblast distribution to an iPSC distribution is a candidate intervention, not just an interpolation. The FM scores each candidate's reachability and its uncertainty; the agent ranks them by expected information gain; and the self-driving lab — robotic transfection plus scRNA-seq, or a perturb-seq screen — *attempts* the top batch. The results do something the in-silico version cannot: they carve the engineerable-versus-forbidden boundary empirically, turning an asserted stratum into a measured one. The existence proof that this works on cells is already published — the "closing the loop" sc-FM work that taught Geneformer from iteratively acquired perturbations and reported a threefold PPV gain on T-cell activation (walked through in the [101 tour](closed-loop-virtual-cells-101.md)) — and **VCHarness** ([Cheng et al., bioRxiv 2026](https://www.biorxiv.org/content/10.64898/2026.04.11.717183v1)) wraps the model-building corner in a multi-agent system. The throughline: the FM is not the answer, it is the cheap simulator that tells the expensive machine where to look.

## The honest failure modes

Four caveats keep this from being oversold, and they compound. First, the **sim-to-real gap** — the FM's density is not the lab's reality, and every round of the loop is a small distribution shift, so a surrogate that was well-calibrated on atlas data drifts as the agent steers experiments into regions the atlas never covered. Second, **surrogate exploitation** — give an agent an acquisition function over a flawed FM and it will find and exploit the FM's errors, proposing experiments that look maximally informative precisely because the model is confidently wrong there; this is the same reward-hacking risk that the rBio reasoning model flagged for FM-as-verifier, transposed to FM-as-simulator. Third, the **throughput ceiling** — the machine is the bottleneck, not the model, so the loop only pays off with *batch* active learning (many diverse experiments per round) and a brutal honesty about how few real cycles a budget actually buys. Fourth, **multiple comparisons at machine speed** — an autonomous loop can generate a hundred plausible-looking hits before a human would notice it is p-hacking, the same hazard the CellVoyager critique raised for autonomous analysis, now with a robot arm attached. And underneath all four sits the obvious one: autonomous biology needs biocontainment and a human gate on what the loop is allowed to physically make.

## Where this goes

The destination is the closed-loop reachability lab. The foundation model proposes which cell states should be engineerable; the self-driving lab attempts them; the failures and successes redraw the realized / engineerable / forbidden boundary as a measured object instead of a claim. That is what finally makes the viability stratification *falsifiable in the world* and not only in silico — you do not argue about whether a state is forbidden, you watch the machine fail to make it, round after round, until the boundary is empirical. The agent without a world model is a blind planner; the foundation model without an oracle is a confident guesser; the machine without a policy is an expensive random walk. Put them in one loop and you get the thing the field has been circling since the first virtual-cell perspective: a system that proposes biology, tests it, and learns from being wrong. The engine is built. What is left is to close the loop on cells the way chemistry already closed it on molecules.
