临床 AI，正是生物学 FM 落地成真产品的地方。一篇研究论文宣称的是一个基准；一次临床部署触碰的是一个患者的诊疗。这是两个世界，成功标准不同、时间线不同、利益相关方也不同。这篇梳理 2025-2026 年的临床部署版图——什么上线了、什么在试点、什么卡在审查里——以及那个越来越能决定「哪项研究真要紧」的部署界面。

## 2026 年的部署界面

今天有五个真实界面，让生物学 FM 触达临床医生和患者：

1. **Claude for Healthcare and Life Sciences**（Anthropic，2026 年 1 月 12 日于 JPM 发布）。带医疗专属护栏的通用 Claude + 一份不断扩充的 MCP 集成 agent 名单。**Owkin Pathology Explorer** 作为第一个重量级生物学 FM agent 跑在里面。其余 K Pro 系列的集成正在按月推出。
2. **Microsoft Dragon Copilot** 以及更广的 Microsoft Healthcare 栈。环境式文档记录、EHR 集成的建议、越来越 agent 化的临床工作流。这是「AI 协助处理病历」这类用例的部署模板。
3. **Hippocratic AI 那批以安全为先的临床 agent**。做出院后照护、用药核对、患者教育的语音 agent。它证明了：安全 + 可靠，可以成为临床 AI 的竞争护城河。
4. **Tempus、Foundation Medicine、Caris**——基因组-病理的多模态栈。FM 辅助的泛癌报告，喂进肿瘤决策支持。
5. **FDA 已批的窄模块**（Paige Prostate、Aidoc、Viz.ai、Caption Health、RapidAI 等）。每个都只为一个特定任务获批；这是大多数临床 FM 产品最终要走的监管模板。

那条结构性的洞见：**护城河是部署界面，不是模型。** 一个稍逊于 SOTA、但干净地接进了 Claude for Healthcare 或 Epic 的模型，触达的患者数，多过一个活在 Jupyter notebook 里的 SOTA 模型。

## 2025-2026 年的头条临床 FM

### DeepRare——带可追溯推理的罕见病诊断

DeepRare（被语料库收为星标）是 2026 年讨论最多的临床 agent 化系统。它把一个 LLM 推理层和对 OMIM、ClinVar、HPO、GeneReviews 以及近期文献的结构化访问拼在一起；在基准的罕见病病例上，它追平专科遗传学家，而且关键在于它**把推理链摆出来**，让临床医生能逐步审查。部署故事还在早期（多数罕见病中心在跑试点），但它是 2026 年最可信的「agent 化临床助手」部署。

### 人类衰老的多模态时钟

2026 年那篇星标论文「Multimodal clocks of human aging」把 EHR + 组学 + 影像 + 功能测量合成一个生物学年龄估计，在多个队列上对照死亡率和发病率终点做了验证。部署界面是预防性诊疗和临床研究；那套数字孪生的框架，正开始吸引正经的投资。

### ECG-FM 及其他生理信号 FM

在医院级 ECG 语料（数百万份 12 导联描记）上训练的基础模型，在心律失常分类、从窦性心律 ECG 检出左室功能障碍、以及早期预警预测上，表现都很强。**视网膜 FM**（RETFound、Moorfields-Google DeepMind 的工作）在眼科里扮演同样的角色——从单张眼底图做预防性疾病检测。

### 病理 agent（另文专述）

Owkin Pathology Explorer、PathChat-2、TITAN 是病理临床 agent 的故事；完整图景见[病理 FM 版图](pathology-fm-landscape.md)。这里值得点一句：Owkin → Claude for Healthcare 的集成，是 2026 年初这个领域最显眼的生物学 FM 临床部署。

### 临床工作流旁的影像 FM

病理之外，**放射 FM**（在 RSNA-bench 上训练的模型、MAIRA-2、RAD-DINO）和**组织-放射多模态模型**，正越来越多地被接进 Epic 和 Cerner。Aidoc 和 Viz.ai 早已演示过部署模板——为卒中检测和肺栓塞检测获批的窄 FDA 模块——而 FM 加持的版本，就是下一代。

## 研究级和临床级，差在哪儿

有六条性质，把一个可部署的生物学 FM 和一个基准冠军区分开来：

1. **分布偏移下的可靠性。** 医院人群各不相同；一个在 Brigham 切片上管用的模型，可能在社区医院切片上失灵。跨机构验证过的稳健性，是入场费。
2. **不确定性的校准。** 一个临床模型说「恶性概率 92%」，那这 92% 在部署里就得真的是 92%。校准失准的 FM 很危险。
3. **可追溯的推理。** DeepRare 那种推理链透明度不是锦上添花；它越来越是一项监管期待。黑箱 FM 会遇逆风。
4. **嵌入既有工作流。** 一个要求临床医生离开 Epic、登进另一个网页应用、再复制粘贴结果的 FM，不会有人用。接进 Epic、Cerner、PACS、LIS——部署的生死就在这儿。
5. **持续监控。** 部署后的 FM，会随患者人群和临床实践演变而悄无声息地退化。在现代框架里，部署后的监控基础设施是强制的。
6. **监管获批路径。** 窄任务走 FDA 510(k)；更宽的工具走 SaMD；欧洲走 EMA 的对应物。路径决定部署速度。

## 评估：临床 RCT vs 回顾性 vs 基于理赔

2026 年的临床 FM 评估有三个层级：

- **临床 RCT**——金标准，但罕见；贵、慢、被监管严格审视。用于最高风险工具的 FDA 批准。
- **回顾性留出**——绝大多数已发表的临床 FM 评估都在这一层。留出一个时间或地域切片，证明模型仍然管用。这是首发论文的标准。
- **基于理赔的结局**——在真实世界理赔数据里，观察部署这个 FM 是否和更好的结局相关。随着 FM 规模化部署，这越来越重要。

[Campanella et al. 2025 的临床基准](evaluation-papers-catalog.md)是最干净的回顾性基准之一；至于生物学 FM 的临床 RCT 证据，2026 年仍然稀缺。

## 小团队能从哪儿切进临床

给一支 1-5 人团队的三个现实 2026 楔子：

- **某个窄专科的 agent 化临床工作流**——拿一个现成 FM（病理、放射、组学）裹上一个 LLM 编排器 + 可追溯推理，瞄准一个病种，和一个临床团队合作。DeepRare 是模板。
- **给罕见病群体的决策支持工具**——小人群被大平台冷落；一个专注的工具能触达它的受众。
- **集成管道**——病理 FM 和 Epic 之间、组学 FM 和 PathOS 之间、agent 化系统和 EHR 之间的连接器。不光鲜，但不可或缺。

作为小团队，*别*干的：试图上线一个 FDA 已批的通用临床 FM。监管成本是数百万美元级；集成成本是数年级。挑一个不和 Owkin、Paige、Microsoft、Anthropic 正面硬碰的楔子。

## 2026-2027 年该盯什么

- **Claude for Healthcare 里下一波 MCP 集成的生物学 agent**——Owkin Pathology Explorer 这个模板，一年内会被基因组 FM、组学、放射的提供方照抄。
- **更多 FDA 已批、FM 加持的窄模块**——预计到 2027 年底，病理、放射、眼科、心脏科里会有 10+ 项新获批。
- **第一项证明「FM 辅助诊疗改善结局」的临床 RCT**——它一落地，监管和报销的版图就会移位。

临床，是科学遇见患者的地方。盯着什么被部署，而不只是什么被刷榜。
