## 那个问题

你能否造出这样一个系统：一个预训练的基础模型（sc-FM、病理 FM 或基因组 FM）坐在中间，一个 LLM 智能体绕着它打转——挑选有代表性的数据集、跑些简单分析、用学到的东西微调 FM、再复检——直到模型在某一个具体临床问题上收敛到有用的东西？具体地说：*药物 X 会在这个胃癌亚组上起效吗？*

这是一个真实的项目形态。它也是大多数智能体在环系统悄悄失败的地方。这篇帖子的要点是在写任何代码之前先把 *为什么* 想清楚。

## 究竟为什么是这个形态

你不是在试图发明一个新架构（那是[小实验室分类法](small-labs-what-to-build-v2.md)里的 Wedge 1）。你是在拿一个冻结的 FM、叠上一个小的任务特定 head，并要求一个 LLM 智能体去充当 **实验规划者**，而不是那个人类研究生。FM 带来生物学预训练。adapter 带来任务特异性。智能体带来 *何时用哪份数据* 的决策。

赌注在于：智能体能比一条固定流水线更好地在数据版图（TCGA、GDSC、PRISM、DepMap、临床队列）里导航，因为正确的下一个数据集取决于模型当前不擅长什么——而智能体能读模型的混淆矩阵，并据此挑选下一轮。

## 写任何代码前你必须决定的五件事

1. **回路的停止规则是什么？** 这是单一最重要的设计决策。"训练损失变平"是错的——模型可以把训练损失推到零，却在一个新患者上仍然无用。"验证 AUC 变平"如果验证集是智能体挑的，那也是错的。诚实的规则：留出一个智能体 **永不可见、永不命名** 的临床队列，并在该队列上的 AUC 连续 N 轮达到平台时停止。
2. **"亚组"究竟是什么意思？** "HER2+ 胃癌"是一种定义；"既往接受过铂类、HER2+ MSI-H 胃癌"是另一种。它越窄，临床上越有用——而任何单一数据集里存在的患者就越少。你将不得不在一个更宽的群体上训练，并在那个窄的群体上 *评估*。
3. **FM 被允许做什么？** 冻结主干 + adapter 是安全的第一步（每轮 1-10 GPU-天）。全量微调是每轮数个 GPU-周，你的回路会在第 3 轮就把算力预算炸掉。除非你有强有力的理由，否则选冻结-加-adapter。
4. **智能体到底决定什么？** 现实的智能体动作：挑下一个数据集、挑下一个亚组切片、挑下一个 adapter 配置、挑对哪个基线作比较。不现实的：发明新模型架构或写新颖的损失函数。把动作空间保持得小而具体。
5. **基线是什么？** 一个作用于简单特征（年龄、突变、表达）上的线性模型，是你必须超越的基线。[线性基线清算](why-linear-baselines-win.md)是整整一年的证据，表明 sc-FM 在扰动任务上输给这个基线。如果你的智能体-加-FM-加-adapter 系统 5 轮后还打不过它，那回路就错了，加更多轮也修不好。

## 最便宜的第一刀

HER2+ 胃癌里的曲妥珠单抗（trastuzumab）。冻结 Virchow2（病理 FM）加一个药物条件 adapter。智能体从 {TCGA-STAD、GDSC 胃癌细胞系、PRISM 胃癌、一个智能体永不触碰的留出内部队列} 中挑各轮。停止规则：留出 AUC 在 3 轮上达到平台。与一个作用于 HER2 IHC + Lauren 分类上的线性逻辑回归作比较。

总预算：约 2-3 万美元算力，约 2 千美元 LLM API 调用，1 人 × 6-12 个月。这小到一个有动力的博士生足以胜任，又具体到那个比较真正有意义。

## 它最可能在哪里失败——保持诚实

- **因果鸿沟。** 你在预测 *如果我们给这个药会发生什么*。你的训练数据是 *接受了该药的患者发生了什么，而这些患者是由肿瘤医生出于临床原因选定的*。那是第一级上的选择偏倚，而[因果模型博客](causal-models-fm-and-vc.md)解释了为什么在没有干预数据或一个结构性假设的情况下，再花哨的建模也修不好它。你要么（a）只在随机试验数据上训练（罕见、量小），要么（b）非常明确地说明这是一个假设生成工具，而非一个临床决策工具。
- **亚组数据稀缺。** 临床亚组越窄，样本越小。在 n=80 上迭代微调到第 2 轮就会过拟合。把亚组用作 *评估者*，而非 *训练者*。
- **收敛剧场。** 智能体回路易于演示、难于复现。智能体会看起来像在"决策"，而其实它是在对提示词的措辞做模式匹配。用不同随机种子把同一个回路跑 5 次；如果最终模型每次都不同，你没有收敛——你有一台老虎机。

## 开始前要读的先行技术

- **Biomni**（Zou lab，2025）——在一个工具库之上的智能体式生物信息学，带一个基准。已发表的、最接近"智能体挑下一项分析"的对应物。
- **TxAgent / TxGemma**（Google，2025）——治疗推理智能体，配一个开放的小型 FM。
- **CellAgent**（2024）——端到端地编排一条 scRNA-seq 流水线。
- **面向扰动的主动学习文献**（Replogle/Theis 2024-2026）——同样的回路想法，应用到 *下一步做哪个敲除*，而非 *加载哪个药物响应数据集*。

## 裁决

这个项目很有趣，楔子也对。风险是真实的，但有名有姓、具体——收敛规则、亚组稀缺、因果鸿沟。它们没有一个是研究终结者；它们是 *设计约束*。如果你让 FM 保持冻结、让留出队列保持密封、先打过线性基线，并把输出标注为一个假设生成工具而非临床决策支持系统，你就有了一个可辩护的 12 个月项目，它能产出一份可发表的结果和一个可复用的代码库。把它从"又一个智能体-LLM 演示"变成真正工作的，是那些约束的纪律，而非回路的巧妙。
