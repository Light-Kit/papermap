## 这个问题

你能不能搭这么一个系统：中间坐着一个预训练好的基础模型（sc-FM、病理 FM，或基因组 FM），外面绕着一个 LLM agent——挑代表性数据集、跑些简单分析、拿学到的东西去微调那个 FM、再回头复检——一圈圈转下去，直到模型在某个具体临床问题上收敛到有用为止？说具体点：*药 X 对这个胃癌亚组管不管用？*

这是个实打实的项目形态。它也正是大多数「agent 在环」系统悄无声息翻车的地方。这篇的意义，就是在动手写任何代码之前，先把*为什么*想清楚。

## 为什么是这个形态

你不是要发明一个新架构（那是[小实验室分类法](small-labs-what-to-build-v2.md)里的楔子 1）。你是拿一个冻结的 FM，叠一个小的任务专属头，再让一个 LLM agent 去当那个**实验规划者**，顶替人类研究生的位置。FM 带来生物学预训练。adapter 带来任务特异性。agent 带来「哪份数据、什么时候用」的决策。

这一注押的是：agent 能比一条固定流水线更会在数据版图里导航（TCGA、GDSC、PRISM、DepMap、临床队列），因为「下一份数据该取哪个」取决于模型当下在哪儿表现差——而 agent 能读模型的混淆矩阵，照着挑下一轮。

## 写任何代码之前必须拍板的五件事

1. **这个环的停机规则是什么？** 这是最要紧的一个设计决定。「训练损失趋平」是错的——模型可以把训练损失压到零，在一个新患者身上仍旧没用。「验证 AUC 趋平」要是验证集是 agent 自己挑的，那也是错的。诚实的规则：封存一个 agent **永远看不到、也永远点不到名**的临床队列，等它在这队列上的 AUC 连续 N 轮趋平再停。
2. **「亚组」到底指什么？** 「HER2+ 胃癌」是一个定义；「有过铂类暴露的 HER2+ MSI-H 胃癌」是另一个。越窄，临床上越有用——而任何单个数据集里的患者也越少。你只能在一个更宽的人群上训练，再到那个窄的上*评估*。
3. **FM 被允许干什么？** 冻结骨干 + adapter 是安全的第一步（每轮 1-10 GPU-天）。整模型微调是每轮 GPU-周，到第 3 轮你的算力预算就被这环炸掉了。没有强理由就选「冻结 + adapter」。
4. **agent 实际上决定什么？** 现实的 agent 动作：挑下一个数据集、挑下一个亚组切片、挑下一套 adapter 配置、挑跟哪条基线比。不现实的：发明新模型架构、写新的损失函数。把动作空间保持得小而具体。
5. **基线是什么？** 一个跑在简单特征（年龄、突变、表达）上的线性模型，是你必须打赢的那条基线。[线性基线那场清算](why-linear-baselines-win.md)是整整一年的证据，说 sc-FM 在扰动任务上输给这条基线。要是你这套 agent + FM + adapter 系统 5 轮之后还赢不了它，那这环就是错的，再加几轮也救不回来。

## 最省的第一刀

HER2+ 胃癌里的曲妥珠单抗。冻结 Virchow2（病理 FM）加一个药物条件化 adapter。agent 从 {TCGA-STAD、GDSC 胃癌细胞系、PRISM 胃癌、一个 agent 永远碰不到的内部留出队列} 里挑每一轮。停机规则：留出 AUC 在 3 轮里趋平。跟一个跑在 HER2 IHC + Lauren 分型上的线性逻辑回归比。

总预算：约 2-3 万美元算力，约 2 千美元 LLM API，1 人 × 6-12 个月。小到一个有心气的博士生就能扛，又具体到这场比较真有意义。

## 它最可能在哪儿翻车——说实话

- **因果缺口。** 你预测的是*如果给这药会怎样*。你的训练数据却是*那些接受了这药的患者后来怎样了，而他们是肿瘤科医生出于临床理由挑出来的*。这是第 1 层上的选择偏倚，[因果模型那篇](causal-models-fm-and-vc.md)讲了：没有干预数据、也没有结构性假设，再花哨的建模都修不好它。你要么（a）只在随机试验数据上训练（罕见、量小），要么（b）非常明确地声明这是个生成假设的工具，不是临床决策工具。
- **亚组数据稀缺。** 临床亚组越窄，样本越小。在 n=80 上做迭代微调，第 2 轮就过拟合了。把亚组当*评估者*，别当*训练者*。
- **收敛剧场。** agent 环很好演示、很难复现。agent 看起来在「决策」，其实是在对提示词的措辞做模式匹配。拿不同随机种子把同一个环跑 5 遍；要是每遍最后的模型都不一样，你没有收敛——你有一台老虎机。

## 动手前要读的前作

- **Biomni**（Zou 组, 2025）——在一个工具库上做 agent 化生物信息学，带基准。最接近「agent 挑下一步分析」的已发表对照物。
- **TxAgent / TxGemma**（Google, 2025）——治疗学推理 agent，配一个开源小 FM。
- **CellAgent**（2024）——端到端编排一条 scRNA-seq 流水线。
- **扰动主动学习那批文献**（Replogle/Theis 2024-2026）——同一个环的想法，只是套在「下一个该敲哪个基因」上，而不是「下一份该加载哪个药物响应数据集」。

## 结论

这项目有意思，楔子也对。风险是真的，但都有名有姓、具体可指——收敛规则、亚组稀缺、因果缺口。它们没一个是研究的死穴；它们是*设计约束*。只要你把 FM 冻住、把留出队列封死、先打赢线性基线、再把产出标成生成假设的工具而非临床决策支持系统，你就有一个守得住的 12 个月项目，能出一篇可发表的结果和一份可复用的代码库。真正把它从「又一个 agent-LLM 演示」变成正经活儿的，是那几条约束的纪律，不是这环本身有多聪明。
