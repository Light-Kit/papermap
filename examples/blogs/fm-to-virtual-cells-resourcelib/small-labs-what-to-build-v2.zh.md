## 为什么这三个楔形能凑到一起

我在此保留的三个楔形都假定同一个起跑位置：你能写 PyTorch，你能交付一个库，而且你有一条愿意为之深度坚持己见的生物学直觉。它们明确地 *不* 假定你能谈成一笔多医院数据交易（在强形式上排除了楔形 4）、能维护一条 10 亿 token 的持续预训练循环（在规模上排除了楔形 3），或能运营一条 FDA 通道的产品线（排除了住在[临床 / 智能体博客](clinical-and-agentic-clinical.md)里的临床部署楔形）。

你能做的——也是 2025-2026 年的记录所显示的、真正从 1-5 人团队里出货的——是发明一个生物学形状的归纳偏置、在别人冻结的检查点之上叠一个微小适配器，或者构建其他人的工具所依赖的胶水。

## 楔形 1——归纳偏置模块

**形态：** 一个新的架构组件或完整骨干，它 *专门为生物学而正确*——而不是一个恰好被改用过来的 transformer。竞争对手不是大 FM 实验室的规模，而是他们对 BERT 的路径依赖。

### 范例

- **[xVERSE (Jiang & Xie, bioRxiv 2026.04)](https://liudengzhang.github.io/conference-vaults/conferences/iclr-2026/tools/)**——转录组原生的非 LM 架构。采用基因-基因协变注意力而非基因-基因序列注意力，把批次感知的归一化融入模型内部而非置于其上游，以及稀疏感知的计算。**结果：** 在表征、批次校正、空间填补上比 scGPT / Geneformer / scFoundation / UCE 提升 11-34%。这是 2026 年最干净的一个数据点，证明 *架构选择在经验上是承重的*。架构拆解另见[架构阶段博客](stage-3-architecture.md)。
- **[TxPert (Wenkel et al., Nat Biotech 2026)](https://doi.org/10.1038/s41587-026-03113-4)**——将一个多知识图先验直接注入 transformer，用于转录组扰动预测。值得注意的是 *谁* 写了它：Wenkel 合著了 2025 年的 `latent-additive` 清算式批评。这位批评者自己的下一步并不是"抛弃 FM"，而是"把朴素 transformer 所缺的知识图先验交给 FM"。这正是楔形 1 的招式——偏置有名字、有论文。
- **scFoundation 的读深感知注意力**——现有的大语料 sc-FM 中最接近架构/生物学协同设计的一个。*那片* 设计空间大多尚未被探索，而那恰恰是一个小团队的生物学功力可以兑现为一篇站得住脚论文的地方。
- **ESM-3 的 SE(3)-不变结构 token**（[ESM-3, Science 2025](https://www.science.org/doi/10.1126/science.ads0018)）——把旋转/平移对称性烤进架构，因为生物学具有这些对称性。这一范式（*挑一个数据具有、而朴素 transformer 忽略的对称性，把它编码进去*）远远超出蛋白质的范畴而通用。
- **AlphaGenome 的轨迹预测目标**（Nature 2026；25/26 项变异效应胜出）——一个基因组 FM，其架构决策是预测 *实验轨迹* 而非核苷酸。目标即归纳偏置。这对基因组数据奏效是因为监督信号（ENCODE/GTEx 轨迹）很丰富；其结构性教训——"让监督塑造架构"——是可移植的。
- **状态空间与长上下文骨干（Hyena / Mamba / Evo2）**——对于上下文长度让 transformer 爆炸的基因组底料，切换到状态空间本身就是那个归纳偏置招式。这里大多数小团队的胜利来自率先把一个已知的 SSM 技巧应用到一个特定生物学语料，而非发明新的 SSM 数学。

### 这需要什么

PyTorch / JAX 的熟练度、对一种生物学底料有主见的解读，以及愿意发表一篇小而尖锐的论文、而非去争一面 SOTA 的旗帜。算力要求不高——上面许多（包括 xVERSE）都是在个位数 GPU-周内训练出来的，因为做功的是架构而非规模。

## 楔形 2——冻结骨干上的小适配器

**形态：** 一个 LoRA / Houlsby 适配器 / 专用头部，通常占骨干参数量的 0.1-5%，用以修复预训练 FM 做不到的一个特定任务。适配范式的完整覆盖在[阶段 5——适配](stage-5-adaptation.md)；这里是真正出过货的人的目录。

### 范例

- **[sc-FM Perturbation Adapter (ICLR 2026 tools)](https://liudengzhang.github.io/conference-vaults/conferences/iclr-2026/tools/sc-fm-perturbation-adapter/)**——一个 <1% 的药物条件适配器，叠在一个冻结的 sc-FM 骨干之上，**击败了线性加性基线**——而后者此前曾在扰动预测上击败每一个全微调的 sc-FM。这是 2026 年被引用最多的单个数据点，说明瓶颈在于预训练目标、而非容量——见[为什么线性基线会赢](why-linear-baselines-win.md)。一个 GPU-周的算力；其余都是迭代纪律。
- **面向扰动任务的药物条件适配器（推广形式）**——Perturbation Adapter 模板自然延伸：敲除条件、剂量条件、时间条件。每一个都是一个独立的小项目，每一个都针对底层 sc-FM 的一种特定失效模式。前沿在于 *你选择注入哪种条件信号*，而非 *你能把头部做多大*。
- **细胞类型专用头部**——用于超出骨干预训练所支持范围的细粒度分类（如 T 细胞亚型分辨）。范式是：冻结一个通用 sc-FM，在你真正关心的细胞类型上训练一个小分类头，把它当作一个带 30 行推理片段的 HuggingFace 模型出货。
- **跨模态适配器**，把 sc-FM 嵌入映射到与成像或空间组学模型共享的空间。Stegle 组关于 SpatialData 感知适配器的工作是 2026 年的参照点；这个楔形大开着，因为几乎还没人桥接过 sc-FM 与空间 FM 的两套栈。
- **ESM-2 功能探针（线性 / 浅层头部）**——大多数最强的"ESM-2 学到了生物学"主张都来自探针研究：在冻结嵌入之上训练一个小头部，问二级结构 / 接触 / 功能是否可解码。每一个这样的探针本质上都是一个可出货的适配器，任何小团队都可以在新蛋白 FM（ESM-3、ProtBERT 的后继者）发布的当周复刻它。
- **病理 FM 的注意力-MIL 聚合器**——病理上的区分度已经 *从骨干移开*（各 ViT 已收敛）*而落到聚合器上*：瓦片级嵌入如何被汇集成切片级预测？CONCH 与 TITAN（Mahmood 实验室）用注意力-MIL；CHIEF 用一个层次化变体。一个能更好处理某个特定病理问题的新聚合器设计——例如用于肿瘤边缘预测的空间上下文感知聚合——就是一篇诚实的小团队论文。

### 这需要什么

PyTorch 熟练度、一个你比任何人都更懂的下游任务，以及在小变体上反复迭代、直到有东西击败线性基线的耐心。算力实打实地是每个实验 1-10 个 GPU-天；区分点是任务知识，而非马力。

## 楔形 6——互操作基础设施

**形态：** *别人的* 工具所依赖的软件——一种数据格式、一个工作流封装器、一座跨语言桥、一个 Galaxy/Bioconductor 适配器。这是会复利的资产类别：一年前的格式被采纳，五年前的格式则成了必然。

### 范例

- **[SpatialData](https://spatialdata.scverse.org/)**（Stegle 组 + 更广的 OME-NGFF 社区）——跨语言、以 Zarr 为后端、用于对齐的空间组学 + 成像 + 注释的容器。空间工具的事实 2026 标准：scverse 采纳了它；Mahmood 实验室的 HEST-1k 用它；主要商业平台（10x Visium HD、Xenium、MERSCOPE、NanoString CosMx）都附带导出适配器。底料的故事见[空间 / 多模态博客](spatial-omics-and-multimodal.md)。
- **plyxp**（Mike Love）——在 Bioconductor 的 `SummarizedExperiment` 上提供 dplyr 风格的动词。小、聚焦、外科手术式。范式是：挑一个跨生态摩擦点（讨厌 Bioconductor S4 API 的 R-tidyverse 用户），构建抚平它的最小桥，维护它十年。
- **bioc-to-galaxy**——LLM 辅助的 Bioconductor → Galaxy XML 转换器。每一个获得 Galaxy 封装的 Bioconductor 工具，都变得能被另一个社区使用（Galaxy 用户有一个工作流构建器，而非 R 控制台）。这是一个 *力倍增* 楔形——它让数百个别人的工具为成千上万的新用户工作，而无需写一个新的分析方法。
- **scverse + scvi-tools + AnnData / MuData**——SpatialData 所栖身的更广生态。大多数基础性工作由小团队完成，他们各自占住一个位置（单细胞容器；深度生成模型实现；多模态扩展）并守住它。如果你处于职业早期、又不想从零出货一个新格式，加入一个 scverse 相邻的维护者团队本身就是一条楔形 6 的路。
- **生物学 FM 的 HuggingFace 移植**——那项不光鲜的活：把一个已发表的检查点拿来，连同妥当的 config、tokenizer 和一个推理示例，作为一个干净的 `AutoModel` 出货。大多数生物学 FM *技术上* 已被其作者 HF 移植过，但在一个全新环境里 `from_pretrained` 的实际体验往往是坏的。无论谁维护那个权威移植，谁就对该 FM 如何被使用拥有结构性权力。
- **面向新模态的 OME-NGFF / Zarr 扩展**——基于 Zarr 的格式扩展面（新维度语义、新元数据 schema）是底料演化之处。单单一个被接受的 OME-NGFF 扩展，就能成为整个下游工具品类的承重件。

### 这需要什么

软件工程纪律（这是未经测试的代码最快现形的楔形），以及对社区 / 治理工作的认真投入——做一个好的开源公民、回应 GitHub issue、参加 scverse 或 OME 会议、撰写让格式真正可被采纳的文档。回报不是引用数；而是你能守住十年的、栈中的结构性位置。

## 把三个楔形叠起来

2025-2026 年诚实的小实验室策略不是"挑一个楔形"。而是在 2-3 年的窗口里 **把这三者中的两个叠起来**：

- **楔形 1 + 楔形 2**——发明出正确的归纳偏置模块 *并* 在其上配上匹配的适配器，于是你对某一个特定下游任务有了一个连贯的故事（架构洞见 + 冻结骨干的部署路径）。xVERSE 加上其上一个 SpatialData 感知的 Perturbation Adapter，就会是一个可信的 2026 序列。
- **楔形 1 + 楔形 6**——发表一个小骨干，*并* 维护它所依赖的互操作底料。scverse 相邻的组（Stegle、Theis）正是这样建立持久位置的。
- **楔形 2 + 楔形 6**——在现有 FM 上出货适配器 *并* 维护分发它们的 HuggingFace / Bioconductor 移植。若你只有一名工程师，这是最不光鲜、却最可守的楔形叠法。

叠起来给你买到的：单个楔形是一篇 1-2 年的论文；两个叠起来的楔形是一个 5 年的研究计划，兼具方法论新颖性（楔形 1 或 2）与持久基础设施（楔形 6）。大 FM 实验室通常不会捡起楔形 6，因为他们的激励是围绕检查点发布、而非格式维护。
