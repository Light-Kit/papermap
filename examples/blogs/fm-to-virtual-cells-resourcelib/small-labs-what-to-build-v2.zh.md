## 这三个切口为什么能凑到一起

我留下来的这三个切口，都假定你站在同一条起跑线上：你会写 PyTorch，能把一个库发出去，而且手里攥着一条你愿意为它较真到底、立场鲜明的生物学直觉。它们**明确不**假定你能去谈一笔跨多家医院的数据交易（这就排除了强版本的切口 4）、能维护一条 10 亿 token 的持续预训练回路（这排除了规模化的切口 3），或者能跑一条 FDA 路线的产品线（这排除了临床部署那个切口，那块归 [临床 / 智能体博客](clinical-and-agentic-clinical.md) 管）。

你**能**做的——也是 2025-2026 年的记录里，1 到 5 人小团队真正交付过的东西——是发明一条生物学形状的归纳偏置，是在别人的冻结 checkpoint 上叠一个极小的适配器，或者去搭那块所有人的工具都得依赖的「胶水」。

## 切口 1——归纳偏置模块

**它长这样：** 一个新的架构组件，或者一整套**专为生物学量身定做**的骨干——而不是一个碰巧被拿来改用途的 transformer。你要拼的不是大 FM 实验室的规模，而是它们身上那种「BERT 路径依赖」。

### 代表作

- **[xVERSE (Jiang & Xie, bioRxiv 2026.04)](https://liudengzhang.github.io/conference-vaults/conferences/iclr-2026/tools/)**——一个转录组原生的非 LM 架构。注意力做的是基因-基因的共变，而不是基因-基因的序列；批次感知的归一化被融进了模型内部，而不是搁在上游；算力也是稀疏感知的。**结果：** 在表征、批次校正、空间填补上，比 scGPT / Geneformer / scFoundation / UCE 高出 11-34%。这是 2026 年最干净的一个数据点，说明**架构选择在经验上是真扛事的**。拆解可参见 [架构阶段博客](stage-3-architecture.md)。
- **[TxPert (Wenkel et al., Nat Biotech 2026)](https://doi.org/10.1038/s41587-026-03113-4)**——把一个多知识图谱的先验直接注进 transformer，去做转录组层面的扰动预测。值得一提的是**它是谁写的**：Wenkel 正是 2025 年那篇 `latent-additive` 清算批评的合著者。这位批评者自己的下一步，不是「抛弃 FM」，而是「把原版 transformer 缺的那个知识图谱先验补给 FM」。这恰恰是切口 1 的标准动作——偏置有名有姓，还有论文。
- **scFoundation 的测序深度感知注意力**——它是现有大语料 sc-FM 里，最接近「架构与生物学协同设计」的一个。可那片设计空间绝大部分还没人探过，而这正是一个小团队的生物学功底能兑现成一篇站得住脚论文的地方。
- **ESM-3 的 SE(3) 不变结构 token**（[ESM-3, Science 2025](https://www.science.org/doi/10.1126/science.ads0018)）——把旋转/平移对称性烤进架构里，因为生物学本来就有这些对称性。这套套路（**挑一个数据有、而原版 transformer 却忽略掉的对称性，把它编码进去**）远不止适用于蛋白质。
- **AlphaGenome 的「轨道预测」目标**（Nature 2026；变体效应 25/26 胜）——一个基因组 FM，它的架构决定是去预测**实验轨道**，而不是核苷酸。**目标本身就是那个归纳偏置。** 这套之所以对基因组数据管用，是因为监督信号（ENCODE/GTEx 轨道）足够丰富；但「让监督去塑造架构」这条结构性的教训，是可以迁移的。
- **状态空间与长上下文骨干（Hyena / Mamba / Evo2）**——对那些上下文长度会把 transformer 撑爆的基因组底料来说，换成状态空间本身就是「上归纳偏置」这步动作。这里小团队的胜绩，大多来自「第一个把某个已知 SSM 技巧用到某个具体生物语料上」，而不是来自发明新的 SSM 数学。

### 这条路需要什么

PyTorch / JAX 要熟，对某一种生物底料要有自己鲜明的判断，还得愿意发一篇「小而扎实」的论文，而不是去抢 SOTA 的旗号。算力要求不高——上面这些（包括 xVERSE）很多都是在个位数的 GPU-周里训出来的，因为真正在干活的是架构，不是规模。

## 切口 2——冻结骨干上的小适配器

**它长这样：** 一个 LoRA / Houlsby 适配器 / 专用头，参数量通常只有骨干的 0.1-5%，专治预训练 FM 干不了的某一个具体任务。适配模式的完整覆盖在 [阶段 5——适配](stage-5-adaptation.md)；这里是一份「谁真的交付了」的名录。

### 代表作

- **[sc-FM 扰动适配器（ICLR 2026 工具）](https://liudengzhang.github.io/conference-vaults/conferences/iclr-2026/tools/sc-fm-perturbation-adapter/)**——一个不到 1% 的「药物条件」适配器，叠在一个冻结的 sc-FM 骨干上，**赢过了那条线性-加性基线**——而后者此前在扰动预测上，是把每一个全量微调的 sc-FM 都按下去过的。这是 2026 年被引最多的那个数据点，说明瓶颈在预训练目标，不在容量大小——见 [为什么线性基线会赢](why-linear-baselines-win.md)。算力一个 GPU-周，剩下的全靠迭代上的纪律。
- **面向扰动任务的药物条件适配器（推广版）**——扰动适配器这套模板天然能往外延：敲除条件、剂量条件、时间条件。每一个都是一个小项目，每一个都对准底层 sc-FM 的某一种具体失效模式。前沿在于**你选哪一路条件信号去注入**，而不是**你能把头做多大**。
- **细胞类型专用头**——做骨干预训练撑不起来的那种细粒度分类（比如 T 细胞亚型的分辨）。套路是：冻住一个通用 sc-FM，在你真正在乎的那种细胞类型上训一个小分类头，再配上一段 30 行的推理代码片段，当成一个 HuggingFace 模型发出去。
- **跨模态适配器**——把 sc-FM 的嵌入映射到一个跟成像或空间组学模型共享的空间里。Stegle 组那套 SpatialData 感知的适配器，是 2026 年的参照点；这个切口大敞着，因为几乎还没人把 sc-FM 和 spatial-FM 两套栈搭起来过。
- **ESM-2 功能探针（线性 / 浅层头）**——「ESM-2 学到了生物学」这类最强的主张，大多出自探针研究：在冻结嵌入之上训一个小头，问一句二级结构 / 接触 / 功能能不能被解码出来。每一个这样的探针，本质上都是一个可交付的适配器，任何小团队都能在某个新蛋白 FM（ESM-3、ProtBERT 的后继者）刚发布那一周就照着复刻一遍。
- **病理 FM 的注意力-MIL 聚合器**——病理领域的差异化已经**从骨干上挪开了**（ViT 已经收敛），**挪到了聚合器上**：tile 级的嵌入怎么汇聚成 slide 级的预测？CONCH 和 TITAN（Mahmood 组）用注意力-MIL；CHIEF 用一个层级化的变体。一个把某个具体病理问题处理得更好的新聚合器设计——比如做肿瘤边缘预测的空间上下文感知聚合——就是一篇老老实实的小团队论文。

### 这条路需要什么

PyTorch 要熟，手里得有一个你比谁都懂的下游任务，再加上耐着性子在小变体上反复磨、直到磨出一个能赢过线性基线的东西。算力是实打实地每个实验 1-10 个 GPU-日；拉开差距的是任务知识，不是马力。

## 切口 6——互操作基础设施

**它长这样：** 一个**别人的**工具都得依赖的软件——一种数据格式、一个工作流封装、一座跨语言的桥、一个 Galaxy/Bioconductor 适配器。这是一类会复利的资产：一个一岁的格式被采用了，一个五岁的格式就成了绕不开的必然。

### 代表作

- **[SpatialData](https://spatialdata.scverse.org/)**（Stegle 组 + 更大的 OME-NGFF 社区）——一个跨语言、以 Zarr 为底的容器，把对齐过的空间组学 + 成像 + 注释装在一起。它是 2026 年空间工具事实上的标准：scverse 采用了它；Mahmood 组的 HEST-1k 用的是它；几大商业平台（10x Visium HD、Xenium、MERSCOPE、NanoString CosMx）也都配了导出适配器。底料的来龙去脉见 [空间 / 多模态博客](spatial-omics-and-multimodal.md)。
- **plyxp**（Mike Love）——在 Bioconductor 的 `SummarizedExperiment` 上套一层 dplyr 风格的动词。小、聚焦、下刀精准。套路是：挑一个跨生态的摩擦点（受不了 Bioconductor S4 API 的 R-tidyverse 用户），搭一座能把这道坎抹平的最小桥，然后维护它十年。
- **bioc-to-galaxy**——一个 LLM 辅助的 Bioconductor → Galaxy XML 转换器。每一个拿到 Galaxy 封装的 Bioconductor 工具，都因此向另一拨社区敞开了门（Galaxy 用户用的是工作流搭建器，不是 R 控制台）。这是一个**力量倍增器**式的切口——它不用新写一个分析方法，就让成百上千个别人的工具，为成千上万的新用户跑了起来。
- **scverse + scvi-tools + AnnData / MuData**——SpatialData 所栖身的那个更大生态。它的大部分基础工作，都是小团队做的：他们各自占住一个位置（单细胞容器；深度生成模型实现；多模态扩展），然后守住。如果你还在职业生涯早期、又不想从零发一个新格式，那加入一个 scverse 周边的维护者团队，本身就是一条切口 6 的路。
- **生物 FM 的 HuggingFace 移植**——一桩不起眼的活儿：把一个发布出来的 checkpoint，做成一个干净的 `AutoModel`，配上像样的 config、tokenizer 和一个推理示例。多数生物 FM **技术上**已经被作者自己 HF-移植过了，但在一个全新环境里真去 `from_pretrained` 的实际体验，往往是坏的。谁来维护那个权威的移植版，谁就对这个 FM 怎么被用握有结构性的权力。
- **面向新模态的 OME-NGFF / Zarr 扩展**——基于 Zarr 的格式扩展面（新的维度语义、新的元数据 schema），正是底料演进的地方。一个被接纳的 OME-NGFF 扩展，就足以成为一整类下游工具的承重梁。

### 这条路需要什么

软件工程上的纪律（这是几个切口里最快暴露出「没测试过的代码」的那一块），外加对社区 / 治理工作真心的投入——做一个好的开源公民，回 GitHub issue，去开 scverse 或 OME 的会，把那些能让格式真正可被采用的文档写出来。回报不是引用数，而是一个你能守住十年的、在某条栈里的结构性位置。

## 把三个切口叠起来

2025-2026 年里，小实验室诚实的打法不是「挑一个切口」，而是在 2-3 年的窗口里，**把这三个里的两个叠起来**：

- **切口 1 + 切口 2**——既发明那个对路的归纳偏置模块，又在它之上配好那个适配器，这样你就有了一个自洽的故事（架构洞见 + 冻结骨干的部署路径），对准某一个具体的下游任务。xVERSE 加上一个 SpatialData 感知的扰动适配器，会是 2026 年一条很可信的序列。
- **切口 1 + 切口 6**——发一个小骨干，**同时**维护它所依赖的那块互操作底料。scverse 周边的几个组（Stegle、Theis）就是这么建起持久位置的。
- **切口 2 + 切口 6**——在现有 FM 上交付适配器，**同时**维护那个分发它们的 HuggingFace / Bioconductor 移植。这套不那么光鲜，但如果你手里只有一个工程师，它是最守得住的切口组合。

叠起来到底买到了什么：单独一个切口是一篇 1-2 年的论文；两个叠起来，就是一个 5 年的研究计划，既有方法上的新意（切口 1 或 2），又有持久的基础设施（切口 6）。大 FM 实验室通常不会去捡切口 6，因为他们的激励是绕着发 checkpoint 转的，不是维护格式。
