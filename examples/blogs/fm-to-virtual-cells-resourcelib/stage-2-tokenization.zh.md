如果你把四篇 FM 方法部分并排着读——scGPT、UNI、ESM-2、AlphaGenome——它们的架构看起来相似多于相异。语料看起来相似多于相异。而 *token 化*，则看起来像是为四个不同问题而生的四项不同发明。token 化是每个家族对 **什么算作生物学的一个单位** 做出承诺的地方，而一旦承诺，这个选择就会向下游每一个阶段传播。本文走过四个家族的 token 化选择，看每一个保留了什么、每一个悄悄丢弃了什么，以及哪些 2026 年的工作正开始打破这些模板。

## 单细胞：从连续计数到离散 token

一个单细胞的转录组是一个 **稀疏的、无序的向量**，含约 20,000 个整数计数。把它变成 transformer 能摄入的东西有两种方式：

- **scGPT 式的分箱。** 每个细胞的表达值按分位数分到 51 个箱里（其他变体用 7-101 个箱）。每个非零基因变成一个 token `(gene_id, bin_id)`；细胞就是这些 token 的序列。词表大小 ≈ 20k × 51 ≈ 1M；实践中 scGPT 把 gene-id 和 bin-id 的嵌入分解开。
- **Geneformer 式的排名。** 在每个细胞内部，按表达对基因排名，并对 *排名位置* 进行 token 化。词表只有约 20k（每个基因一个 token），而"表达"信息完全活在 token 顺序里。

两种选择都编码了该领域承重的归纳偏置——*在这个细胞里，哪些基因相对基线表达最高*——代价是丢弃两样东西：**绝对幅度**（某基因 1000 UMI vs 10 UMI，在排名或粗分箱之后看起来相同）和 **基因-基因顺序**（基因没有标准序列；位置嵌入本质上是噪声）。UCE 和 CellPLM 加了跨物种条件 token；scFoundation 加了读深感知 token；**xVERSE**（Jiang & Xie，bioRxiv 2026.04）完全放弃了语言模型框架，使用转录组原生的非 LM token，这也是它在表征任务上以 11-34% 击败每个 sc-FM 的部分原因。

## 病理：ViT 块作为事实标准

病理 FM 直接从计算机视觉复用了一个 token 化配方：一张全切片图像（WSI）在一个固定放大倍数（通常 20×，有时 40×）下被切成 **256×256 像素的块**（有时 224×224，偶尔 512×512）。每个块被喂给一个 **Vision Transformer（ViT）**，它内部把块再切成 14×14 或 16×16 的子块，每个子块线性投影成一个 token 嵌入。词表是无界的（每个块是它自己的嵌入，没有离散码本）；位置由块内的 2D 学习位置嵌入编码，并在聚合阶段由块的切片坐标编码。

它丢弃了什么：**尺度不变性**（模型只在一个放大倍数下看块——要在多个放大倍数下查询，你需要多尺度金字塔）、**块间上下文**（在 FM 阶段每个块是独立的；空间上下文只在[适配](stage-5-adaptation.md)的聚合 / attention-MIL 阶段才到来），以及 **颜色校准**（染色归一化在 FM 上游；如果它做得差，FM 不会补偿）。Mahmood 实验室的 UNI、Paige 的 Virchow2 和 HistAI 的 Hibou 全都用本质上相同的 ViT 块配方；差异在于预训练目标和语料，而不在 token 化。

## 蛋白质：字母表已经为你定好了

蛋白质享有一种其他家族都没有的 token 化简洁性：**生物学已经供给了一个 20-token 的词表**（20 种标准氨基酸），外加用于硒代半胱氨酸、吡咯赖氨酸、间隙、未知的特殊 token。ESM-2 和大多数蛋白 FM 直接使用这个原始的 20+特殊字母表。ESM-3 通过加入 **结构 token**（一个针对局部 3D 环境的离散 VQ-VAE 码本）和 **功能 token**（GO 术语、InterPro 结构域）作为并行模态进行泛化，它们全都共享同一个 transformer。现代蛋白设计模型（RFdiffusion、Chroma）使用氨基酸字母表，但作用在一个连续的几何骨架上，而非一个离散序列上。

对蛋白 FM *真正重要* 的那个 token 化选择，是要 **跨字母表共享参数**，还是把每个氨基酸当作完全独立的。ESM-2 的绑定嵌入 + 旋转位置编码竟然捕获了出人意料大量的进化信号——这也是为什么 ESM-2 表征的迁移性比任何单细胞嵌入迄今所能做到的都更远。

## 基因组：token 化之战尚未结束

基因组 FM 是这样一个家族：在 2026 年，token 化 **仍在被积极争论**。有四条路线活着：

- **k-mer token**（DNABERT-2、NT-V2）：把 3-6 个核苷酸合并成一个 token。词表保持小（4³=64 到 4⁶=4096）；上下文长度保持可控。代价：k-mer 边界是任意的，会扰乱基序对齐。
- **带海量上下文的单核苷酸 token**（HyenaDNA 长达 1M 碱基，Evo2 长达 1M+ 碱基）：每个核苷酸都是它自己的 token；负担转移到架构上（状态空间模型取代注意力；见[第 3 阶段](stage-3-architecture.md)）。
- **针对核苷酸的 BPE**（Caduceus、某些 Evo 变体）：像 GPT-2 对文本那样学一个 token 化器。给出适应基序频率的变长 token；代价是你无法轻易把变异对齐回单碱基坐标。
- **调控轨迹感知的 token**（AlphaGenome 2026）：序列被 token 化为核苷酸块，但用染色质可及性、CpG 甲基化和保守性的并行输入通道加以增强。这正是在 2026 年赢下 25/26 条变异效应轨迹的选择。

基因组 FM 领域之所以没有趋同，是因为正确的 token 化取决于下游任务：变异效应预测想要单核苷酸分辨率；调控元件预测能容忍 k-mer；长程基因调控预测则得益于 BPE。

## 逃逸路线：xVERSE 与后 token 化时代

依我们之见，2026 年关于 token 化最有趣的工作，是 **干脆完全不再 token 化**——至少在 LM 的意义上。xVERSE 的非 LM 转录组原生架构（见[会议库](https://liudengzhang.github.io/conference-vaults/conferences/iclr-2026/tools/)）不假装一个细胞是一个句子。它直接在连续的计数矩阵上操作，用为正确归纳偏置（基因-基因协变、批次效应、稀疏性）设计的注意力模块。相对 sc-FM 基线的 11-34% 增益，一部分是架构故事（在[第 3 阶段](stage-3-architecture.md)中讲述），一部分是 *token 化* 故事——那些 BERT 克隆的 sc-FM 全都在 token 化阶段付出了代价，而 xVERSE 干脆不付这笔代价。我们预期这个模式会重演：随着该领域消化线性基线清算，会有更多家族重新审视它们从 NLP 继承来的"离散 token LM"框架。

token 化是你对自己数据的世界观做出承诺的地方。仔细挑——在适配时你修不好它。
